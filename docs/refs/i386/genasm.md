# genasm.md

対象ファイル: `i386/genasm.c`

## 概要

GDT (Global Descriptor Table) のバイナリデータをアセンブリソース (`table.s`) として出力するホストツール。カーネルのビルド時にホスト上で実行され、GDT エントリをバイト列の `.byte` ディレクティブとして生成する。カーネル自体のコードではなく、ビルドシステムの一部である。

生成された `table.s` は 16 ビットコードとしてアセンブルされ、カーネルのブートシーケンスで GDT として使用される。

## 定数・マクロ

### genasm.c

| 定数 | 値 | 説明 |
|------|------|------|
| `A_ASM` | `"table.s"` | 出力ファイル名 |

### 386.h (セグメントタイプ)

| 定数 | 値 | 説明 |
|------|------|------|
| `ST_CODE` | `0x9a` | コードセグメント (実行可能・読み取り可能・Accessed ビット付き) |
| `ST_DATA` | `0x92` | データセグメント (読み書き可能・Accessed ビット付き) |
| `ST_STACK` | `0x96` | スタックセグメント (読み書き可能・伸長方向下・Accessed ビット付き) |
| `ST_TSS` | `0x89` | TSS セグメント (genasm.c では未使用、386.c で使用) |
| `ST_CALL` | `0x84` | コールゲート (genasm.c では未使用) |

### 386.h (ビット幅指定)

| 定数 | 値 | 説明 |
|------|------|------|
| `_16BIT` | `0x00` | 16 ビットセグメント (D/B=0, G=0) |
| `_32BIT` | `0xc0` | 32 ビットセグメント (D/B=1, G=1: リミットは 4KB 粒度) |

### 386.h (ゲートタイプ)

| 定数 | 値 | 説明 |
|------|------|------|
| `GT_INTR` | `0x8e` | 割り込みゲート (genasm.c では未使用) |
| `GT_TRAP` | `0x8f` | トラップゲート (genasm.c では未使用) |
| `GT_CALL` | `0x8c` | コールゲート (genasm.c では未使用) |

## 構造体・型

### seg_t (386.h)

```c
typedef struct seg {
    unsigned short  limit_l;    /* リミット下位 16 ビット */
    unsigned short  base_l;     /* ベースアドレス下位 16 ビット */
    unsigned char   base_m;     /* ベースアドレス中位 8 ビット (ビット 16-23) */
    unsigned char   type;       /* タイプ + DPL + P ビット */
    unsigned char   limit_h;    /* リミット上位 4 ビット + G/D/L/AVL フラグ */
    unsigned char   base_h;     /* ベースアドレス上位 8 ビット (ビット 24-31) */
} seg_t;
```

x86 GDT/LDT セグメントディスクリプタの 8 バイト構造体。`set_gdt()` でフィールドを設定し、`gen_asm()` でバイト列として出力される。

### gate_t (386.h)

```c
typedef struct gate {
    unsigned short  offset_l;   /* オフセット下位 16 ビット */
    unsigned short  sel;        /* セグメントセレクタ */
    unsigned char   count;      /* パラメータカウント (コールゲート用) */
    unsigned char   type;       /* ゲートタイプ + DPL + P ビット */
    unsigned short  offset_h;   /* オフセット上位 16 ビット */
} gate_t;
```

x86 ゲートディスクリプタの 8 バイト構造体。genasm.c では直接使用されない (IDT 設定は `386.c` の `set_gate()` で行われる)。

## 関数リファレンス

### set_gdt

```c
void set_gdt(short n, unsigned long base, unsigned long limit,
             unsigned char type, unsigned char dpl, unsigned char gda);
```

**概要:** GDT エントリを設定する。386.c 版と同一のロジック。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `n` | `short` | セレクタ値 (8 の倍数、例: 0x08, 0x20)。内部で 8 で割ってインデックスに変換 |
| `base` | `unsigned long` | セグメントベースアドレス (32 ビット) |
| `limit` | `unsigned long` | セグメントリミット (20 ビット) |
| `type` | `unsigned char` | セグメントタイプ (`ST_CODE`, `ST_DATA`, `ST_STACK`) |
| `dpl` | `unsigned char` | 特権レベル (0 = Ring 0, 3 = Ring 3) |
| `gda` | `unsigned char` | 粒度/デフォルトサイズフラグ (`_16BIT` = 0x00, `_32BIT` = 0xc0) |

**戻り値:** なし

**処理内容:**

1. セレクタ値を 8 で割ってインデックスに変換: `n /= 8`
2. ベースアドレスを 3 フィールドに分割して格納:
   - `base_l` = 下位 16 ビット
   - `base_m` = ビット 16-23
   - `base_h` = ビット 24-31
3. リミットを 2 フィールドに分割して格納:
   - `limit_l` = 下位 16 ビット
   - `limit_h` = 上位 4 ビット + `gda` フラグ
4. タイプフィールドに DPL を合成: `type | (dpl << 5)`

**呼び出し元:** `main()`

**注意点:**
- `type` フィールドには Present ビット (bit 7) が含まれている前提。`ST_CODE` = 0x9a は P=1 が設定済み。
- DPL は `type` のビット 5-6 に格納される: `type | (dpl << 5)`

### gen_asm

```c
void gen_asm(void);
```

**概要:** GDT のバイナリデータをアセンブリソースファイル (`table.s`) として出力する。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. `table.s` をライトモードでオープン
2. ヘッダコメントとアセンブリディレクティブを出力:
   - `.code16` (16 ビットコード)
   - `.text` セクション
   - `.globl start` (エントリポイント)
3. GDT データを 256 バイト (32 エントリ x 8 バイト) 分ループ:
   - 8 バイトごとに `.byte 0xNN, 0xNN, ...` 形式で出力
4. `.org 4096` でファイルを 4096 バイトにパディング
5. ファイルクローズ

**呼び出し元:** `main()`

**注意点:**
- 出力は GDT のみ。IDT は含まれない (IDT はカーネル起動時に `386.c` / `interrupt.c` で動的に設定される)。
- `.org 4096` により、生成されたバイナリは正確に 4KB となる。これはメモリアドレス `AL_GDT` (0x2000) からの配置に対応する。

### main

```c
int main(void);
```

**概要:** GDT エントリを初期化し、アセンブリソースを生成する。

**引数:** なし

**戻り値:** 暗黙的に 0 (return 文なし)

**処理内容:**

以下の GDT エントリを `set_gdt()` で設定する:

| セレクタ | ベース | リミット | タイプ | DPL | ビット幅 | 説明 |
|----------|--------|----------|--------|-----|----------|------|
| `0x08` | `0x3000` | `0xffff` | `ST_CODE` | 0 | 16 | 16 ビットカーネルコード (AL_KERNEL16) |
| `0x10` | `0xb8000` | `0xffff` | `ST_DATA` | 0 | 16 | 16 ビットカーネルデータ (VGA VRAM) |
| `0x18` | `AL_KERNEL16` | `0x0` | `ST_STACK` | 0 | 16 | 16 ビットカーネルスタック |
| `SEL_K32_C` (0x20) | `0x0` | `0xfffff` | `ST_CODE` | 0 | 32 | 32 ビットカーネルコード |
| `SEL_K32_D` (0x28) | `0x0` | `0xfffff` | `ST_DATA` | 0 | 32 | 32 ビットカーネルデータ |
| `SEL_K32_S` (0x30) | `0x0` | `0x0` | `ST_STACK` | 0 | 32 | 32 ビットカーネルスタック |
| `SEL_U32_C` (0x58) | `0x0` | `0xfffff` | `ST_CODE` | 3 | 32 | 32 ビットユーザーコード |
| `SEL_U32_D` (0x60) | `0x0` | `0xfffff` | `ST_DATA` | 3 | 32 | 32 ビットユーザーデータ |
| `SEL_U32_S` (0x68) | `0x0` | `0x0` | `ST_STACK` | 3 | 32 | 32 ビットユーザースタック |

設定後、`gen_asm()` を呼び出してアセンブリソースを生成する。

**呼び出し元:** ビルドシステム (Makefile)

**注意点:**
- TSS エントリ (`SEL_TSS0`, `SEL_TSS1`, `SEL_TSS_DUMMY0`, `SEL_TSS_DUMMY1`) と syscall ゲート (`SEL_SYSCALL`) は genasm.c では設定されない。これらはカーネル起動時に `386.c` の `set_gdt()` / `set_gate()` で動的に設定される。
- 16 ビットデータセグメント (0x10) のベースアドレスが `0xb8000` (VGA テキスト VRAM) に設定されている。16 ビットモードから VGA に直接アクセスするため。
- 32 ビットセグメントはフラットモデル (ベース 0、リミット 0xFFFFF × 4KB = 4GB) で設定されている。

## 補足

### ビルドフロー

```
genasm.c → (ホストコンパイル) → genasm → (実行) → table.s → (アセンブル) → table.o
```

1. `genasm.c` をホストの gcc でコンパイル
2. 生成された `genasm` 実行バイナリを実行
3. `table.s` が生成される (GDT のバイナリデータを `.byte` ディレクティブで記述)
4. `table.s` を GAS でアセンブルして `table.o` を生成
5. `table.o` はカーネルバイナリにリンクされ、メモリアドレス 0x2000 に配置される

### GDT エントリの構造

x86 GDT セグメントディスクリプタは 8 バイトで以下のレイアウトを持つ:

```
ビット    63       56 55  52 51  48 47       40 39       32
         +----------+------+------+----------+----------+
         | Base H   | G|D|L| Lim H|  Type    |  Base M  |
         +----------+------+------+----------+----------+
ビット    31       16 15        0
         +----------+----------+
         | Base L   | Limit L  |
         +----------+----------+
```

- G (Granularity): 0 = バイト粒度、1 = 4KB 粒度
- D/B (Default): 0 = 16 ビット、1 = 32 ビット
- Type: P (Present) + DPL + S + タイプビット
