# elf.md

対象ファイル: `i386/elf.c`

## 概要

ELF (Executable and Linkable Format) バイナリローダ。ホスト上で実行されるビルドツールであり、カーネル自体のコードではない。リンカが出力した ELF 形式のカーネルバイナリからプログラムヘッダに基づいてセグメントを抽出し、フラットバイナリ (生のメモリイメージ) として出力する。

コードセグメント (PF_R|PF_X) とデータセグメント (PF_R|PF_W) を連続したバイナリとして結合し、QEMU にロード可能なカーネルイメージを生成する。

## 定数・マクロ

### ELF 標準定数 (elf.h)

| 定数 | 説明 |
|------|------|
| `PT_LOAD` | ロード可能セグメント。メモリに配置されるセグメントを示す |
| `PF_R` | 読み取り可能フラグ |
| `PF_W` | 書き込み可能フラグ |
| `PF_X` | 実行可能フラグ |

## 構造体・型

### Elf32_Ehdr (elf.h)

ELF ヘッダ。ファイルの先頭に配置され、プログラムヘッダテーブルのオフセット (`e_phoff`)、エントリ数 (`e_phnum`)、エントリサイズ (`e_phentsize`) を含む。

### Elf32_Phdr (elf.h)

プログラムヘッダ。各セグメントの種類 (`p_type`)、フラグ (`p_flags`)、仮想アドレス (`p_vaddr`)、ファイルオフセット (`p_offset`)、ファイルサイズ (`p_filesz`)、メモリサイズ (`p_memsz`) を記述する。

## グローバル変数

| 変数名 | 型 | サイズ | 説明 |
|--------|------|--------|------|
| `buf[640000]` | `char` | 640KB | セグメントデータの読み込みバッファ。カーネルサイズは 640KB 以下に制限される |

## 関数リファレンス

### write_space

```c
void write_space(int fd_d, int len);
```

**概要:** 出力ファイルに指定バイト数のパディング (空白領域) を書き込む。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `fd_d` | `int` | 出力ファイルディスクリプタ |
| `len` | `int` | パディングバイト数 |

**戻り値:** なし

**処理内容:**

1. `lseek(fd_d, len - 1, SEEK_CUR)` で出力ファイルのオフセットを `len - 1` バイト前進
2. `writen(fd_d, buf, 1)` で 1 バイトを書き込み (ファイルサイズを確定させる)
3. デバッグ出力でパディングサイズを表示

**呼び出し元:** `elf()`

**注意点:**
- `lseek` + 1 バイト書き込みによる「穴あき」ファイル作成。ファイルシステムがスパースファイルをサポートしていればディスク上の実サイズは小さくなる。
- ローカル変数 `buf[2]` でグローバルの `buf` を隠蔽している (未初期化の 1 バイトが書き込まれる)。

### elf

```c
void elf(int fd_s, int fd_d, Elf32_Word p_type, Elf32_Word p_flags,
         Elf32_Addr* addr, int* size, int flag);
```

**概要:** ELF ファイルから指定タイプ・フラグのセグメントを検索し、フラットバイナリとして出力する。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `fd_s` | `int` | 入力 ELF ファイルのファイルディスクリプタ |
| `fd_d` | `int` | 出力フラットバイナリのファイルディスクリプタ |
| `p_type` | `Elf32_Word` | 検索するセグメントタイプ (`PT_LOAD`) |
| `p_flags` | `Elf32_Word` | 検索するセグメントフラグ (`PF_R|PF_X` または `PF_R|PF_W`) |
| `addr` | `Elf32_Addr*` | [入出力] 前回出力セグメントの仮想アドレス / マッチしたセグメントの仮想アドレス |
| `size` | `int*` | [入出力] 前回出力セグメントのメモリサイズ / マッチしたセグメントのメモリサイズ |
| `flag` | `int` | 0 = 最初のセグメント (パディングなし)、1 = パディング挿入あり |

**戻り値:** なし

**処理内容:**

1. 入力ファイルを先頭にリワインド
2. ELF ヘッダ (`Elf32_Ehdr`) を読み取り
3. プログラムヘッダテーブルにシークし、全プログラムヘッダを読み取り
4. 全プログラムヘッダのデバッグ情報を表示 (タイプ、フラグ、仮想アドレス、メモリサイズ、ファイルサイズ、オフセット)
5. `p_type` と `p_flags` が一致するセグメントを検索:
   - 完全一致、または `p_flags` が `PF_R|PF_X` で実際のフラグが `PF_R|PF_W|PF_X` の場合もマッチ
6. マッチしたセグメント:
   - `flag == 1` の場合: `write_space()` で前セグメント末尾からのギャップをパディング
   - セグメントデータを読み込み (`readn`)、出力ファイルに書き込み (`writen`)
   - `addr` と `size` を更新

**呼び出し元:** `main()`

**注意点:**
- `PF_R|PF_X` (コード) マッチ時に `PF_R|PF_W|PF_X` も許容する特別なロジックがある。これは一部のリンカが実行可能コードに書き込み可能フラグも付ける場合に対応するため。
- 最初にマッチしたセグメントのみを処理する (`break` で抜ける)。
- `e_ph[32]` は `static` 宣言されており、複数回の呼び出し間でデータが保持される。

### main

```c
int main(int argc, char* argv[]);
```

**概要:** コマンドライン引数で指定された ELF ファイルからコードとデータのセグメントを抽出し、フラットバイナリを出力する。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `argc` | `int` | コマンドライン引数の数 |
| `argv` | `char*[]` | コマンドライン引数配列。`argv[1]` = 入力 ELF、`argv[2]` = 出力バイナリ |

**戻り値:** なし (`exit()` で終了)

**処理内容:**

1. 引数が 3 未満ならエラー終了
2. 入力ファイル (`argv[1]`) と出力ファイル (`argv[2]`) をオープン
3. 第 1 回 `elf()` 呼び出し: `PT_LOAD` + `PF_R|PF_X` (コードセグメント) を抽出 (flag=0: パディングなし)
4. 第 2 回 `elf()` 呼び出し: `PT_LOAD` + `PF_R|PF_W` (データセグメント) を抽出 (flag=1: パディングあり)
5. `exit(0)` で終了

**呼び出し元:** ビルドシステム (Makefile)

**注意点:**
- 使用方法: `elf <入力ELFファイル> <出力バイナリファイル>`
- コードセグメントが先に出力され、続いてギャップ (パディング) を挟んでデータセグメントが出力される。
- 出力ファイルは `O_RDWR | O_CREAT` でオープンされるが、パーミッションモード引数が省略されている。

### readn

```c
int readn(int fd, char* buf, int len);
```

**概要:** 指定バイト数を完全に読み込む。部分的な `read()` を繰り返して全バイトを読み取る。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `fd` | `int` | ファイルディスクリプタ |
| `buf` | `char*` | 読み込みバッファ |
| `len` | `int` | 読み込みバイト数 |

**戻り値:**

| 値 | 説明 |
|------|------|
| `-1` | エラー |
| (なし) | 成功時は return 文がない (未定義動作) |

**処理内容:**

1. `len > 0` の間ループ:
   - `read(fd, buf, len)` で部分読み込み
   - エラー (ret < 0) なら -1 を返す
   - バッファポインタと残りバイト数を更新

**呼び出し元:** `elf()`

**注意点:**
- 成功時に明示的な return 文がない。コンパイラ依存の未定義動作となる。
- `read()` が 0 を返す場合 (EOF) の処理がなく、無限ループになる可能性がある。

### writen

```c
int writen(int fd, char* buf, int len);
```

**概要:** 指定バイト数を完全に書き込む。部分的な `write()` を繰り返して全バイトを書き込む。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `fd` | `int` | ファイルディスクリプタ |
| `buf` | `char*` | 書き込みバッファ |
| `len` | `int` | 書き込みバイト数 |

**戻り値:**

| 値 | 説明 |
|------|------|
| `-1` | エラー |
| (なし) | 成功時は return 文がない (未定義動作) |

**処理内容:**

1. `len > 0` の間ループ:
   - `write(fd, buf, len)` で部分書き込み
   - エラー (ret < 0) なら -1 を返す
   - バッファポインタと残りバイト数を更新

**呼び出し元:** `elf()`, `write_space()`

**注意点:**
- `readn()` と同様に、成功時の return 文がない。

## 補足

### ビルドフロー

```
kernel.elf (リンカ出力)
  → elf kernel.elf kernel (本ツール実行)
    → kernel (フラットバイナリ)
      → QEMU にロード
```

### ELF セグメント構成

典型的なカーネル ELF ファイルは以下の PT_LOAD セグメントを持つ:

| セグメント | フラグ | 内容 |
|-----------|--------|------|
| コード | `PF_R\|PF_X` | .text セクション (実行可能コード) |
| データ | `PF_R\|PF_W` | .data / .bss セクション (初期化済み/未初期化データ) |

### 出力バイナリの構造

```
+------------------+ 仮想アドレス (コード)
| コードセグメント |
+------------------+
| パディング       | (仮想アドレスギャップを埋める)
+------------------+ 仮想アドレス (データ)
| データセグメント |
+------------------+
```

- コードとデータの間のギャップは `write_space()` でゼロ埋め (lseek) される
- 出力バイナリはメモリ上の配置をそのまま反映したフラットイメージとなる

### 既知の問題

- `readn()` / `writen()` の成功時に return 文がない
- `open()` の `O_CREAT` 使用時にパーミッションモード引数が欠けている
- バッファサイズ 640KB の制限があり、それ以上のカーネルサイズには対応できない
- b-free プロジェクトの `build_boot2.c` を参考に書かれたコード
