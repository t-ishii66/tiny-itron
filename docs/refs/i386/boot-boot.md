# boot/boot.s

対象ファイル: `i386/boot/boot.s`

## 概要

BIOS ブートローダー。フロッピーディスクの先頭セクタ (512 バイト) に配置される第一段ブートローダーである。BIOS によりアドレス 0x7C00 にロードされた後、自身を 0x70000 にコピーして再配置し、フロッピーディスクからカーネルイメージをメモリ上 (0x200:0000 = 物理アドレス 0x2000 以降) に読み込む。読み込み完了後、far jump で start.s (0x300:0000 = 物理アドレス 0x3000) へ制御を移す。

ファイル末尾オフセット 510 に BIOS ブートシグネチャ 0xAA55 を配置し、BIOS がブート可能セクタとして認識できるようにしている。

## 定数・マクロ

| 名前 | 値 | 説明 |
|------|------|------|
| `b_base` | `0x07C0` | BIOS がブートセクタをロードするセグメントアドレス (0x07C0:0000 = 物理 0x7C00) |
| `m_base` | `0x7000` | ブートローダーの再配置先セグメントアドレス (0x7000:0000 = 物理 0x70000) |
| `kt_base` | `0x0200` | カーネル読み込み先の初期セグメントアドレス (0x0200:0000 = 物理 0x2000) |
| `s_base` | `0x0300` | セカンドブート (start.s) のセグメントアドレス (0x0300:0000 = 物理 0x3000) |
| `delta_seg` | `0x1000` | セグメントローテーション時の加算値 (64KB 境界超え対策、0x1000 セグメント = 64KB) |

## ラベル・ルーチンリファレンス

### start

```asm
start:
    movw  $b_base, %ax
    movw  %ax, %ds
    movw  %ax, %ss
    movw  $1024, %sp
```

**概要:** ブートローダーのエントリポイント。BIOS により物理アドレス 0x7C00 にロードされた直後に実行される。

**処理内容:**

1. DS と SS を `b_base` (0x07C0) に設定し、SP を 1024 (0x0400) に設定する。スタックは 0x07C0:0400 = 物理 0x8000 付近に配置される
2. ブートセクタ 512 バイトを `m_base` (0x7000:0000 = 物理 0x70000) にコピーする (`rep movsb`)
3. `ljmp $m_base, $jumphere` で再配置先へ far jump し、以降はコピー先のコードを実行する

**呼び出し元:** BIOS (電源投入時のブートシーケンス)

**注意点:** 自己再配置を行うことで、元の 0x7C00 付近のメモリ領域をカーネルロード時に上書き可能にしている。

### jumphere

```asm
jumphere:
    movw  $m_base, %ax
    movw  %ax, %ds
    movw  %ax, %es
    movw  %ax, %ss
    movw  $1024, %sp
```

**概要:** 再配置後のエントリポイント。再配置先でセグメントレジスタを再設定する。

**処理内容:**

1. DS, ES, SS を `m_base` (0x7000) に設定し、SP を 1024 に設定する
2. `puts` でブートメッセージ ("Booting TinyITRON 1.0.0 Copyright (c) 2000 by t-ishii66") を表示する
3. `init_fd` でフロッピーディスクコントローラのパラメータを初期化する
4. `load` でカーネルイメージをフロッピーから読み込む
5. "Second Boot ..." メッセージを表示する
6. `ljmp $0x300, $0` でカーネルの start.s (0x300:0000 = 物理 0x3000) へ far jump する

**呼び出し元:** `start` から far jump

### puts

```asm
puts:
    pushw  %bp
    movw   %sp, %bp
    pushw  %bx
    pushw  %cx
    movw   4(%bp), %bx
```

**概要:** NULL 終端文字列を BIOS INT 10h でコンソールに出力する。

**引数:** スタック経由で文字列ポインタ (16 ビットオフセット) を受け取る。

**処理内容:**

1. 引数で渡された文字列ポインタを取得する
2. 1 文字ずつ `putc` を呼び出して出力する
3. NULL (0x00) に到達したら終了する

**呼び出し元:** `jumphere`, `readsect`

### putc

```asm
putc:
    pushw  %bx
    movb   $0x0e, %ah
    movb   $2, %bl
    int    $0x10
    popw   %bx
    ret
```

**概要:** 1 文字を BIOS INT 10h (AH=0x0E: テレタイプ出力) で画面に表示する。

**引数:** AL レジスタに出力する文字コードを設定しておく。

**処理内容:**

1. AH=0x0E (テレタイプ出力), BL=2 (表示色: 緑) を設定する
2. INT 10h を発行して文字を出力する

**呼び出し元:** `puts`

### load

```asm
load:
    movw  $kt_base, %ax
    movw  %ax, %es
    movw  $0x1, %ax
    movw  $0x0000, %bx
load_1:
    ...
```

**概要:** フロッピーディスクからカーネルイメージをメモリに読み込むメインループ。

**処理内容:**

1. ES を `kt_base` (0x0200) に設定し、読み込み先アドレスを 0x0200:0000 (物理 0x2000) にする
2. 論理セクタ番号 1 から開始し (セクタ 0 はブートセクタ自体)、`readsect` で 1 セクタずつ読み込む
3. 読み込みごとに BX を 512 バイト加算し、論理セクタ番号 AX をインクリメントする
4. BX が 0xFFFF を超えてラップアラウンドした場合 (`testw $0xffff, %bx` で検出)、ES を `delta_seg` (0x1000 = 64KB) 加算してセグメントローテーションする
5. 論理セクタ番号が 300 に達したら読み込み完了 (セクタ 1〜299 の 299 セクタ、299 × 512 = 153,088 bytes ≒ 150KB のカーネルイメージ)

**呼び出し元:** `jumphere`

**注意点:** 読み込みサイズは 299 セクタ (約 150KB) に固定されている。カーネルサイズが変わった場合はこの値を調整する必要がある。

### readsect

```asm
readsect:
    pushw  %bp
    movw   %sp, %bp
    movw   4(%bp), %ax
    movb   $36, %dl
    divb   %dl
```

**概要:** 論理セクタ番号から CHS (シリンダ・ヘッド・セクタ) アドレスを計算し、BIOS INT 13h (AH=02h) でフロッピーディスクから 1 セクタを読み込む。

**引数:** スタック経由で論理セクタ番号 (16 ビット) を受け取る。

**処理内容:**

1. 論理セクタ番号を CHS に変換する:
   - `論理セクタ / 36` の商 = シリンダ番号、余り = ヘッド * 18 + セクタ
   - 余りを `/ 18` で割り、商 = ヘッド番号、余り = セクタ番号
   - セクタ番号は 1 ベースのため `incb %cl` で +1 する
2. INT 13h を発行する (AH=02h: セクタ読み込み, AL=1: 1 セクタ, DL=0: ドライブ A:)
3. 読み込み進捗をスピナー文字 (`-\|/`) で表示する。キャリーフラグがセットされた場合は "Read Sector Error" を表示する

**呼び出し元:** `load` (load_1 ループ内)

**注意点:**
- 1.44MB フロッピーディスクを前提とした CHS パラメータ (18 セクタ/トラック, 2 ヘッド) を使用する
- シリンダ番号の上位 2 ビットは無視される (コメントに "warning" あり)

### init_fd

```asm
init_fd:
    pushw  %es
    movw   $0x78, %bx
    movw   $0, %ax
    movw   %ax, %es
    movw   $fd144, %es:(%bx)
    movw   %ds, %es:2(%bx)
    popw   %es
    ret
```

**概要:** フロッピーディスクコントローラのパラメータテーブルを設定する。

**処理内容:**

1. BIOS データ領域のフロッピーパラメータテーブルポインタ (0x0000:0x0078、INT 1Eh ベクタ) を、ブートローダー内の `fd144` テーブルに書き換える
2. `fd144` は 1.44MB フロッピーディスク用のパラメータテーブル (11 バイト) である

**呼び出し元:** `jumphere`

### loop0

```asm
loop0:
    jmp  loop0
```

**概要:** 無限ループ。far jump 後は到達しないが、安全のために配置されている。

## データ定義

| 名前 | 内容 | 説明 |
|------|------|------|
| `fd144` | `0xaf, 2, 25, 2, 36, 0x1b, 0xff, 0x54, 0xf6, 15, 8` | 1.44MB フロッピーディスクパラメータテーブル |
| `secondboot` | `"\r\nSecond Boot ...\r\n"` | カーネル遷移前メッセージ |
| `message` | `"Booting TinyITRON 1.0.0 Copyright (c) 2000 by t-ishii66\r\n"` | 起動メッセージ |
| `readerr` | `"Read Sector Error\r\n"` | 読み込みエラーメッセージ |
| `dot0`, `dot1` | バックスペース + スピナー文字 | 進捗表示用 |
| `dot_data` | `"-\|/"` | スピナーのアニメーション文字 |
| `dot` | `.word 0x0000` | スピナーカウンタ |

## 補足

### メモリマップ (ブート時)

```
0x00000 - 0x01FFF  BIOS データ領域
0x02000 - 0x020FF  GDT (table.s)
0x02100 - 0x028FF  IDT
0x03000 - 0x033FF  start.s (16 ビットカーネル)
0x03400 -          run.s + カーネル本体 (32 ビット)
0x07C00 - 0x07DFF  ブートセクタ (BIOS ロード先、後に再配置)
0x70000 - 0x701FF  ブートセクタ再配置先
```

### ブートシーケンス

1. BIOS が 0x7C00 にブートセクタをロード
2. `start` → 自身を 0x70000 にコピー → `jumphere` へ far jump
3. フロッピーからセクタ 1〜299 を 0x2000 以降に読み込み
4. `ljmp $0x300, $0` で start.s へ遷移

### ブートシグネチャ

```asm
.org  510
.word 0xaa55
```

オフセット 510-511 に BIOS ブートシグネチャ 0xAA55 を配置。BIOS はこのシグネチャを確認してブート可能セクタと判定する。
