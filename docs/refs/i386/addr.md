# addr.md

対象ファイル: `i386/addr.h`

## 概要

メモリアドレスマップとセグメントセレクタの定数定義。カーネルの線形アドレス配置、GDT セレクタ値、CPU ごとのスタックポインタ、メモリプール範囲、および FDC バッファアドレスを定義する。

本カーネルはフラットメモリモデル (ベースアドレス 0) を採用しつつ、16 ビット起動コードと 32 ビットカーネル/ユーザーコードのセグメントを GDT で管理する。

## 定数・マクロ

### 線形アドレス

| 定数 | 値 | 説明 |
|------|------|------|
| `AL_BOOT` | `0x7c00` | BIOS ブートローダのロードアドレス |
| `AL_KERNEL16` | `0x3000` | 16 ビットカーネルコードの開始アドレス。4KB ページ境界に配置 |
| `AL_KERNEL32` | `0x0` | 32 ビットカーネルコードのベースアドレス (フラットモデル) |
| `AL_USER32` | `0x0` | 32 ビットユーザーコードのベースアドレス (フラットモデル、カーネルと共通) |
| `AL_GDT` | `0x2000` | GDT (Global Descriptor Table) の配置アドレス |
| `AL_IDT` | `0x2100` | IDT (Interrupt Descriptor Table) の配置アドレス |
| `AL_MAX` | `0x500000` | 使用可能メモリの上限 (5MB) |

### セグメントオフセットアドレス

| 定数 | 値 | 説明 |
|------|------|------|
| `AS_GDT` | `0` | GDT のセグメントオフセット (16 ビットモード基準) |
| `AS_IDT` | `0` | IDT のセグメントオフセット (16 ビットモード基準) |

### セグメントセレクタ

#### 16 ビットカーネルセグメント

| 定数 | 値 | RPL | 説明 |
|------|------|-----|------|
| `SEL_K16_C` | `0x08` | 0 | 16 ビットカーネルコードセグメント |
| `SEL_K16_D` | `0x10` | 0 | 16 ビットカーネルデータセグメント |
| `SEL_K16_S` | `0x18` | 0 | 16 ビットカーネルスタックセグメント |

#### 32 ビットカーネルセグメント (Ring 0)

| 定数 | 値 | RPL | 説明 |
|------|------|-----|------|
| `SEL_K32_C` | `0x20` | 0 | 32 ビットカーネルコードセグメント。CS=0x20 |
| `SEL_K32_D` | `0x28` | 0 | 32 ビットカーネルデータセグメント。DS=0x28 |
| `SEL_K32_S` | `0x30` | 0 | 32 ビットカーネルスタックセグメント。SS=0x30 |

#### TSS (Task State Segment) セレクタ

| 定数 | 値 | 説明 |
|------|------|------|
| `SEL_TSS0` | `0x38` | CPU 0 の TSS セレクタ。`ltr` で Task Register にロードし、TSS.esp0 を動的更新してタスクスイッチを実現 |
| `SEL_TSS1` | `0x40` | CPU 1 の TSS セレクタ。CPU 0 と同様に `ltr` + TSS.esp0 動的更新方式 |

#### 32 ビットユーザーセグメント (Ring 3)

| 定数 | 値 | RPL | 説明 |
|------|------|-----|------|
| `SEL_U32_C` | `0x58` | 3 | 32 ビットユーザーコードセグメント。CS=0x5B (0x58 | RPL 3) |
| `SEL_U32_D` | `0x60` | 3 | 32 ビットユーザーデータセグメント。DS=0x63 (0x60 \| RPL 3) |
| `SEL_U32_S` | `0x68` | 3 | 32 ビットユーザースタックセグメント。SS=0x6B (0x68 | RPL 3) |

#### システムコールゲート

| 定数 | 値 | 説明 |
|------|------|------|
| `SEL_SYSCALL` | `0x70` | システムコール用コールゲートセレクタ |

### CPU 0 スタックポインタ

| 定数 | 値 | 説明 |
|------|------|------|
| `CPU0_SP` | `0x7a0000` | CPU 0 の初期スタックポインタ (Ring 0 起動時) |
| `CPU0_SP0` | `0x790000` | CPU 0 の Ring 0 スタックポインタ (TSS の SP0 に設定) |
| `CPU0_SP3` | `0x780000` | CPU 0 の Ring 3 スタックポインタ (TSS の SP3 に相当) |

### CPU 1 スタックポインタ

| 定数 | 値 | 説明 |
|------|------|------|
| `CPU1_SP` | `0x770000` | CPU 1 の初期スタックポインタ (Ring 0 起動時) |
| `CPU1_SP0` | `0x760000` | CPU 1 の Ring 0 スタックポインタ (TSS の SP0 に設定) |
| `CPU1_SP3` | `0x750000` | CPU 1 の Ring 3 スタックポインタ (TSS の SP3 に相当) |

### メモリプール

| 定数 | 値 | 説明 |
|------|------|------|
| `STACK_START` | `0x700000` | ユーザータスク用スタックプールの開始アドレス |
| `STACK_END` | `0x74ffff` | ユーザータスク用スタックプールの終了アドレス (320KB) |
| `MEM_START` | `0x110000` | ユーザーメモリプールの開始アドレス。カーネルメモリプール (kmem_alloc) の終端でもある |
| `MEM_END` | `0x6fffff` | ユーザーメモリプールの終了アドレス (約 6MB) |

### FDC バッファ

| 定数 | 値 | 説明 |
|------|------|------|
| `FDC_BUFFER` | `0x10000` | FDC DMA 転送用バッファアドレス (64KB)。ISA DMA の 16MB 制約内 |

### Per-Task カーネルスタック

| 定数 | 値 | 説明 |
|------|------|------|
| `KERN_STACK_SIZE` | `4096` | 1 タスクあたりのカーネルスタックサイズ (4KB) |
| `KERN_STACK_BASE` | `0x750000` | カーネルスタック領域のベースアドレス。タスク N のスタックトップ = `KERN_STACK_BASE + (N+1) * KERN_STACK_SIZE` |

## 補足

### メモリマップ全体像

```
0x00000000 +---------------------------+
           | (予約領域)               |
0x00002000 | GDT (AL_GDT)             |
0x00002100 | IDT (AL_IDT)             |
0x00003000 | 16 ビットカーネル         |
           | (AL_KERNEL16)            |
0x00007C00 | ブートローダ (AL_BOOT)    |
0x00010000 | FDC DMA バッファ          |
           | (FDC_BUFFER)             |
0x00020000 | カーネルメモリプール       |
           | (kmem_alloc, ~960KB)     |
           | Supervisor ページ        |
0x00100000 +---------------------------+ 1MB
           |                           |
0x00110000 | ユーザーメモリプール      |
           | (MEM_START - MEM_END)    |
0x00500000 | AL_MAX (レガシー定数、    |
           |  実際のメモリ使用量は    |
           |  これを超える)           |
0x006FFFFF +---------------------------+
0x00700000 | ユーザースタックプール    |
           | (STACK_START-STACK_END)  |
0x0074FFFF +---------------------------+
0x00750000 | CPU1 Ring 3 スタック      |
0x00760000 | CPU1 Ring 0 スタック      |
0x00770000 | CPU1 初期スタック         |
0x00780000 | CPU0 Ring 3 スタック      |
0x00790000 | CPU0 Ring 0 スタック      |
0x007A0000 | CPU0 初期スタック         |
           +---------------------------+
```

### GDT エントリ一覧

| インデックス | セレクタ | 用途 |
|--------------|----------|------|
| 0 | 0x00 | NULL ディスクリプタ |
| 1 | 0x08 | 16 ビットカーネルコード |
| 2 | 0x10 | 16 ビットカーネルデータ (VGA VRAM 0xB8000 ベース) |
| 3 | 0x18 | 16 ビットカーネルスタック |
| 4 | 0x20 | 32 ビットカーネルコード (Ring 0) |
| 5 | 0x28 | 32 ビットカーネルデータ (Ring 0) |
| 6 | 0x30 | 32 ビットカーネルスタック (Ring 0) |
| 7 | 0x38 | TSS0 (CPU 0) |
| 8 | 0x40 | TSS1 (CPU 1) |
| 9 | 0x48 | (未使用) |
| 10 | 0x50 | (未使用) |
| 11 | 0x58 | 32 ビットユーザーコード (Ring 3) |
| 12 | 0x60 | 32 ビットユーザーデータ (Ring 3) |
| 13 | 0x68 | 32 ビットユーザースタック (Ring 3) |
| 14 | 0x70 | syscall コールゲート |

### セレクタ値と RPL

ユーザーモード (Ring 3) では、セレクタ値に RPL (Requested Privilege Level) ビットが付加される:
- `CS = SEL_U32_C | 3 = 0x58 | 0x03 = 0x5B`
- `SS = SEL_U32_S | 3 = 0x68 | 0x03 = 0x6B`

### スタック配置

各 CPU のスタックは 64KB 間隔で配置されている。スタックは x86 の慣例に従い下方向に伸長する:
- CPU 0: 0x7A0000 → 下方向
- CPU 1: 0x770000 → 下方向
