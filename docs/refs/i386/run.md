# run.s

対象ファイル: `i386/run.s`

## 概要

32 ビットプロテクトモードのエントリポイント。start.s からの far jump (`ljmp $0x20, $0x3400`) によりリニアアドレス 0x3400 に配置されたこのコードに制御が渡される。データセグメント (DS, ES, FS, GS) とスタックセグメント (SS) をカーネル用 32 ビットセレクタに設定し、グローバル変数 `cpu_num` を確認して BSP (CPU 0) と AP (CPU 1) のパスに分岐する。それぞれの CPU 用スタックを設定した後、C 言語の `main()` 関数を呼び出す。

## ラベル・ルーチンリファレンス

### run

```asm
run:
    cli
    mov  $0x28, %ax
    mov  %ax, %ds
    mov  %ax, %es
    mov  %ax, %fs
    mov  %ax, %gs
    mov  $0x30, %ax
    mov  %ax, %ss
    cmpl $0, cpu_num
    jne  ap_entry
    movl $0x07a0000, %esp
    cli
    call main
    jmp  halt
```

**概要:** 32 ビットプロテクトモードのエントリポイント。BSP と AP の共通初期化を行い、CPU 番号に応じて分岐する。

**処理内容:**

1. `cli` で割り込みを禁止する
2. DS, ES, FS, GS を 0x28 (SEL_K32_D: 32 ビットカーネルデータセグメント、ベース 0x0、リミット 4GB) に設定する
3. SS を 0x30 (SEL_K32_S: 32 ビットカーネルスタックセグメント) に設定する
4. グローバル変数 `cpu_num` を確認する:
   - `cpu_num == 0`: BSP パス。ESP を 0x7A0000 (CPU0_SP) に設定し、`main()` を呼び出す
   - `cpu_num != 0`: AP パス (`ap_entry` へジャンプ)
5. `main()` からリターンした場合は `halt` ラベルの無限ループに入る

**呼び出し元:** start.s (far jump `ljmp $0x20, $0x3400`)

**注意点:**

- `cli` を 2 回呼び出している (エントリ時と `call main` の直前)。2 回目の `cli` は安全のための冗長な呼び出しである
- `cpu_num` は外部変数 (`.extern cpu_num`) として参照される。BSP の初期起動時は 0 であり、AP 起動前に BSP が 1 に設定する

### ap_entry

```asm
ap_entry:
    movl  $0x0770000, %esp
    cli
    call  main
```

**概要:** AP (CPU 1) のエントリポイント。AP 専用のスタックを設定して `main()` を呼び出す。

**処理内容:**

1. ESP を 0x770000 (CPU1_SP) に設定する。BSP とは異なるスタック領域を使用し、スタック競合を防ぐ
2. `cli` で割り込みを禁止する
3. `main()` を呼び出す。`main()` 内で `cpu_num` を確認し、AP 用の初期化パス (`smp_ap_init()`) に入る

**呼び出し元:** `run` ラベル内で `cpu_num != 0` の場合

**注意点:** `main()` からリターンした場合は `halt` ラベルの無限ループに入る。

### halt

```asm
halt:
    jmp  halt
```

**概要:** 無限ループ。`main()` からリターンした場合のフォールバック。

**処理内容:** HLT 命令ではなく JMP による無限ループ。CPU は常にビジー状態になる。

## データ定義

### stack_ptr

```asm
.data
.align  4
stack_ptr:
    .long  0x0070000    # offset
    .word  0x30          # selector
```

**概要:** スタックポインタのデータ定義。現在のコードでは直接参照されていない (レガシー)。

| フィールド | 値 | 説明 |
|-----------|------|------|
| オフセット | 0x0070000 | スタックオフセット |
| セレクタ | 0x30 | SEL_K32_S (32 ビットカーネルスタックセグメント) |

## 補足

### BSP と AP のスタック配置

| CPU | スタックトップ | 定数名 | 備考 |
|-----|-------------|--------|------|
| CPU 0 (BSP) | 0x7A0000 | CPU0_SP | 約 7.6MB 地点 |
| CPU 1 (AP) | 0x770000 | CPU1_SP | 約 7.4MB 地点 |

各 CPU のスタックは十分なギャップ (0x30000 = 192KB) を持って配置されている。スタックは下方向に伸長するため、CPU 0 のスタックは 0x7A0000 から下へ、CPU 1 のスタックは 0x770000 から下へ成長する。

### セレクタ一覧

| セレクタ | 値 | 名前 | 用途 |
|---------|------|------|------|
| 0x28 | SEL_K32_D | 32 ビットカーネルデータ | DS, ES, FS, GS に設定 |
| 0x30 | SEL_K32_S | 32 ビットカーネルスタック | SS に設定 |

### SMP 起動フロー

1. BSP: `start.s` → `run.s` (run) → `cpu_num == 0` → ESP = 0x7A0000 → `main()`
2. BSP の `main()` 内で `smp_init()` が AP に SIPI を送信し、`cpu_num` を 1 に設定する
3. AP: `start.s` (再実行) → `run.s` (run) → `cpu_num != 0` → `ap_entry` → ESP = 0x770000 → `main()`
4. AP の `main()` 内で `smp_ap_init()` が呼ばれ、AP 固有の初期化を行う
