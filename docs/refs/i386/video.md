# video.c / video.s / video.h / videoP.h

対象ファイル: `i386/video.c`, `i386/video.s`, `i386/video.h`, `i386/videoP.h`

## 概要

VGA テキストモードドライバ。6845 CRT コントローラ互換の VGA テキストモード (80x25) を操作し、カーネルメッセージの表示とスクロールを行う。以下のファイルで構成される:

- **video.c**: VGA ドライバの C 言語実装。`printk` (カーネル printf)、文字出力、スクロール、直接 VGA 書き込み関数を提供する
- **video.s**: `video.c` の GCC コンパイル出力 (手書きアセンブリではない)。レガシーファイルとして残されている
- **video.h**: パブリックヘッダ。外部から呼び出し可能な関数のプロトタイプ宣言
- **videoP.h**: プライベートヘッダ。VGA 定数、静的変数、内部関数のプロトタイプ、カスタム va_list マクロの定義

## 定数・マクロ

### videoP.h: VGA 定数

| 名前 | 値 | 説明 |
|------|------|------|
| `G_BASE` | `0xB8000` | VGA テキストモードのリニアアドレス (ビデオメモリの先頭) |
| `G_WIDTH` | `80 * 2` (= 160) | 1 行のバイト数 (80 文字 x 2 バイト/文字) |
| `G_HEIGHT` | `26` | 画面の行数 (スクロール用バッファとして 26 行を使用) |
| `G_TOTAL` | `G_WIDTH * G_HEIGHT` (= 4160) | 画面全体のバイト数 |
| `G_VID_ORG` | `12` | 6845 レジスタ番号: 表示開始アドレス (Start Address High/Low、レジスタ 12-13) |
| `G_CURSOR` | `14` | 6845 レジスタ番号: カーソル位置 (Cursor Location High/Low、レジスタ 14-15) |
| `G_ATTR` | `0x02` | 文字属性: 緑色文字 + 黒背景 (前景: 緑 = ビット 1、背景: 黒 = 0) |

### io.h: I/O ポート定数 (関連)

| 名前 | 値 | 説明 |
|------|------|------|
| `IO_6845` | `0x3D4` | 6845 CRT コントローラのインデックスレジスタ |
| `IO_6845_V` | `0x3DA` | 入力ステータスレジスタ 1 (垂直帰線検出用) |

### videoP.h: カスタム可変長引数マクロ

```c
typedef char*  va_list;
#define va_start(ap, param)   (ap = (char*)&param + sizeof(param))
#define va_start2(ap, param)  (ap = (char*)param + sizeof(param))
#define va_arg(ap, type)      ((type*)(ap += sizeof(type)))[-1]
#define va_end(ap)
```

**概要:** 標準ライブラリの `<stdarg.h>` を使用せず、独自の可変長引数マクロを定義している。フリースタンディング環境 (OS カーネル) では標準ライブラリが利用できないため、スタック上の引数配置に基づいた独自実装を使用する。

| マクロ | 説明 |
|--------|------|
| `va_start(ap, param)` | `param` の次の引数アドレスを `ap` に設定する。`&param` のアドレスを基に計算する |
| `va_start2(ap, param)` | `param` がポインタの場合の `va_start` 変形版。ポインタの値自体をベースアドレスとして使用する。`printk2` で使用される |
| `va_arg(ap, type)` | `ap` を `sizeof(type)` バイト進め、直前の位置にある `type` 型の値を返す |
| `va_end(ap)` | 空マクロ (何もしない) |

## グローバル変数

### video_lk (video.c)

```c
static unsigned long video_lk = 0;
```

**概要:** ビデオ出力用スピンロック変数。`printk` および `printk2` の排他制御に使用する。SMP 環境で複数 CPU からの同時出力を防止する。

**初期値:** 0 (アンロック状態)

**使用箇所:** `printk()`, `printk2()` の開始時に `smp_lock(&video_lk)` でロック、終了時に `smp_unlock(&video_lk)` でアンロックする。

### c_x (videoP.h)

```c
static unsigned short c_x;
```

**概要:** 現在のカーソル X 座標 (列番号、0-79)。

### c_y (videoP.h)

```c
static unsigned short c_y;
```

**概要:** 現在のカーソル Y 座標 (行番号、0-24)。

### c_y_max (videoP.h)

```c
static unsigned short c_y_max;
```

**概要:** 現在の Y 座標の最大値。`video_init()` で 24 に初期化される。スクロール時にインクリメントされる。

### scrolltop (videoP.h)

```c
static unsigned short scrolltop;
```

**概要:** スクロールオフセット。6845 の表示開始アドレスレジスタに設定される値。1 行スクロールすると 80 が加算される (80 文字/行)。`scrolltop` が `80 * 25` を超えた場合、VGA メモリの先頭にラップアラウンドする。

## 関数リファレンス

### video_init

```c
void video_init(void);
```

**概要:** ビデオドライバを初期化する。カーソル位置をリセットし、6845 の表示開始アドレスを設定する。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. `c_x` と `c_y` を 0 に設定する (カーソルを左上に移動)
2. `c_y_max` を 24 に設定する
3. `scrolltop` を 0 に設定する
4. `video_set_6845(G_VID_ORG, 0)` で 6845 の表示開始アドレスを 0 にリセットする

**呼び出し元:** カーネル初期化 (`main()` 内の `all_init()`)

### printk

```c
void printk(char *s, ...);
```

**概要:** カーネル用 printf 関数。フォーマット文字列と可変長引数を受け取り、VGA テキストモードに出力する。スピンロックによる排他制御付き。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `s` | `char *` | フォーマット文字列 |
| `...` | 可変長 | フォーマット指定子に対応する引数 |

**戻り値:** なし

**処理内容:**

1. `smp_lock(&video_lk)` でスピンロックを取得する
2. フォーマット文字列を 1 文字ずつ処理する:
   - `%s`: `va_arg` で `char*` を取得し、`video_puts` で文字列出力
   - `%x`: `va_arg` で `long` を取得し、`video_putn(n, 16)` で 16 進数出力
   - `%d`: `va_arg` で `long` を取得し、`video_putn(n, 10)` で 10 進数出力
   - `%c`: `va_arg` で `long` を取得し、`video_putc` で 1 文字出力
   - その他の `%` 後の文字: `%` と次の文字をそのまま出力
   - `%` 以外の文字: `video_putc` でそのまま出力
3. `smp_unlock(&video_lk)` でスピンロックを解放する

**呼び出し元:** カーネル全体 (デバッグ出力、初期化メッセージ、エラー出力など)

**注意点:**

- **ユーザープロセスから呼び出してはならない**。スクロール時の I/O 操作があるため、Ring 3 での呼び出しは安全ではない
- `%` フォーマット指定子は `%s`, `%x`, `%d`, `%c` のみ対応。`%p`, `%u`, `%ld` 等は未サポート
- 負の整数の 10 進数表示は未サポート (符号なし整数として扱われる)

### printk2

```c
void printk2(char **sp);
```

**概要:** システムコールハンドラ用の printk 変形版。フォーマット文字列のポインタをポインタ経由で受け取る。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `sp` | `char **` | フォーマット文字列へのポインタのポインタ。`*sp` がフォーマット文字列を指す |

**戻り値:** なし

**処理内容:**

1. `*sp` からフォーマット文字列のポインタを取得する
2. `va_start2(ap, sp)` で可変長引数の開始位置を設定する (`va_start2` は `sp` の値をベースアドレスとして使用)
3. 以降は `printk` と同じフォーマット処理を行う
4. スピンロックによる排他制御付き

**呼び出し元:** `c_intr_syscall` (システムコールハンドラ) 内の printk システムコール

**注意点:** `va_start2` マクロは `va_start` と異なり、`param` のアドレス (`&param`) ではなく `param` の値そのものをベースアドレスとして使用する。これはシステムコールの引数がスタック上に直接積まれているためである。

### video_puts

```c
void video_puts(char *p);
```

**概要:** NULL 終端文字列を VGA に出力する。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `p` | `char *` | 出力する NULL 終端文字列 |

**戻り値:** なし

**処理内容:**

1. 文字列の各文字に対して `video_putc` を呼び出す
2. `c_y == 20` の場合、各文字の後に '#' を追加出力する (デバッグ用、スクロール領域検出)
3. NULL 文字に到達したら終了する

**呼び出し元:** `printk`, `printk2`, `video_putn`

**注意点:** `c_y == 20` のチェックは、スタティック変数 `c` の値 ('#') を追加出力するデバッグ用コードが残っている。

### video_putc

```c
void video_putc(char c);
```

**概要:** 1 文字を VGA テキストモードに出力する。改行処理、行末折り返し、画面スクロールを自動的に行う。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `c` | `char` | 出力する文字 |

**戻り値:** なし

**処理内容:**

1. 出力位置のアドレスを計算する: `G_BASE + 2 * scrolltop + c_y * 160 + c_x * 2`
2. 文字が `'\n'` の場合:
   - `c_y` をインクリメントし、`c_x` を 0 にリセットする
3. それ以外の場合:
   - `video_wait()` を呼び出す (現在は空実装)
   - 文字コードを書き込む (偶数バイト)
   - 属性バイト `G_ATTR` (0x02: 緑) を書き込む (奇数バイト)
   - `c_x` をインクリメントする
   - `c_x` が 80 以上になった場合、`c_x` を 0 にリセットし `c_y` をインクリメントする
4. `c_y` が 24 を超えた場合、`video_scroll()` を呼び出して画面をスクロールし、`c_y` を 24 に設定する

**呼び出し元:** `video_puts`, `printk`, `printk2`

### video_putn

```c
void video_putn(unsigned int n, int mode);
```

**概要:** 数値を文字列に変換して VGA に出力する。16 進数と 10 進数をサポートする。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `n` | `unsigned int` | 出力する数値 (符号なし) |
| `mode` | `int` | 出力モード: 16 = 16 進数、10 = 10 進数 |

**戻り値:** なし

**処理内容:**

1. `n == 0` の場合、"0" を出力して終了する
2. `mode == 16` の場合:
   - 下位バイトから順に 2 桁ずつ 16 進数文字に変換する
   - 変換済み文字を先頭に挿入して逆順を維持する
   - `video_puts` で出力する
3. `mode == 10` の場合:
   - `n % 10` で下位桁から順に 10 進数文字に変換する
   - 変換済み文字を先頭に挿入して逆順を維持する
   - `video_puts` で出力する

**呼び出し元:** `printk`, `printk2` (フォーマット指定子 `%x`, `%d`)

**注意点:**

- 16 進数出力では小文字 (`a`-`f`) を使用する
- バッファサイズは 64 バイト (十分な余裕がある)
- 16 進数出力はバイト単位 (2 桁ずつ) で処理するため、上位の不要なゼロバイトは出力されない (例: 0x00FF は "ff")

### video_set_6845

```c
static void video_set_6845(unsigned short reg, unsigned short val);
```

**概要:** 6845 CRT コントローラのレジスタに 16 ビット値を書き込む。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `reg` | `unsigned short` | レジスタ番号 (高位バイト用)。`reg` と `reg+1` の 2 つのレジスタに書き込む |
| `val` | `unsigned short` | 書き込む 16 ビット値 |

**戻り値:** なし

**処理内容:**

1. `video_wait()` で垂直帰線を待つ (現在は空実装)
2. `IO_6845` (0x3D4) にレジスタ番号 `reg` を書き込む
3. `IO_6845 + 1` (0x3D5) に `val` の上位バイトを書き込む
4. `IO_6845` にレジスタ番号 `reg + 1` を書き込む
5. `IO_6845 + 1` に `val` の下位バイトを書き込む

**呼び出し元:** `video_init`, `video_scroll`, `video_cursor`, `vga_clear`

### video_scroll

```c
static void video_scroll(void);
```

**概要:** 画面を 1 行上方向にスクロールする。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. `scrolltop` を 80 加算する (80 文字/行)
2. `c_y_max` をインクリメントする
3. `video_set_6845(G_VID_ORG, scrolltop)` で 6845 の表示開始アドレスを更新する (ハードウェアスクロール)
4. `video_clear_line()` で新しい最下行をクリアする
5. `scrolltop > 80 * 25` (= 2000) の場合 (VGA メモリのスクロール可能範囲を超えた場合):
   - VGA メモリの下部から先頭にデータをコピーする (`video_copy`)
   - カーソル位置とスクロールオフセットをリセットする
   - 再度 `video_clear_line()` で最下行をクリアする

**呼び出し元:** `video_putc` (`c_y > 24` の場合)

**注意点:** ハードウェアスクロールは 6845 の表示開始アドレスを変更するだけなので高速だが、VGA メモリサイズの制限からラップアラウンド処理が必要になる。

### video_clear_line

```c
static void video_clear_line(void);
```

**概要:** 現在の行 (c_y) を NUL 文字でクリアする。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. 現在のカーソル位置 (`c_y`, `c_x`, `scrolltop`) からアドレスを計算する
2. 160 バイト (80 文字 x 2 バイト) を 0x00 で埋める

**呼び出し元:** `video_scroll`

### video_cursor

```c
static void video_cursor(void);
```

**概要:** 6845 のハードウェアカーソル位置を更新する。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. カーソル位置を計算する: `c_y * 160 + c_x`
2. `video_set_6845(G_CURSOR, c)` で 6845 のカーソルレジスタに設定する

**呼び出し元:** (現在のコードでは呼び出されていない)

**注意点:** 定義はあるが、`video_putc` などから明示的に呼び出されていないため、ハードウェアカーソルは更新されない。

### video_wait

```c
static void video_wait(void);
```

**概要:** 垂直帰線期間を待機する関数。現在は空実装。

**引数:** なし

**戻り値:** なし

**処理内容:** 何もしない (コメントアウトされた実装は `IO_6845_V` (0x3DA) のステータスレジスタをポーリングして垂直帰線を待つものだった)。

### vga_clear

```c
void vga_clear(void);
```

**概要:** VGA テキスト画面全体をクリアする (直接 VGA メモリ書き込み)。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. VGA メモリ (0xB8000) の先頭から 80x25 文字 (= 2000 ワード) に `0x0700 | ' '` (灰色背景 + スペース) を書き込む
2. `scrolltop` を 0 にリセットする
3. `c_x` と `c_y` を 0 にリセットする

**呼び出し元:** `draw_header()` (ユーザータスク、Ring 3)

**注意点:**
- `printk` のスクロール機能で使用する `G_ATTR` (0x02: 緑) とは異なり、`0x07` (灰色) で初期化する。
- 以前は `video_set_6845(G_VID_ORG, 0)` で表示開始アドレスをリセットしていたが、`video_set_6845` は `outb` (I/O ポート命令) を使用するため、Ring 3 (IOPL=0) から呼び出すと #GP 例外が発生する。`video_init` (Ring 0) で既に表示開始アドレスは 0 に設定されているため、ここでは省略している。

### vga_write_at

```c
void vga_write_at(int row, int col, char *s, unsigned char attr);
```

**概要:** 指定された行・列に文字列を直接書き込む。スクロール不要の固定位置表示に使用する。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `row` | `int` | 表示行番号 (0-24) |
| `col` | `int` | 表示列番号 (0-79) |
| `s` | `char *` | 出力する NULL 終端文字列 |
| `attr` | `unsigned char` | 文字属性バイト |

**戻り値:** なし

**処理内容:**

1. VGA メモリ上の指定位置のアドレスを計算する: `G_BASE + row * 80 + col` (ワード単位)
2. 文字列の各文字に対して、文字コードと属性バイトを組み合わせたワード (`attr << 8 | char`) を書き込む
3. 列番号が 80 に達するか NULL 文字に到達したら終了する

**呼び出し元:** ユーザータスク (ステータス表示など)

**注意点:** スクロールオフセット (`scrolltop`) を考慮しない直接書き込みであるため、スクロールが発生した後は表示位置がずれる可能性がある。Ring 3 (ユーザーモード) からも安全に呼び出し可能。

### vga_write_dec_at

```c
void vga_write_dec_at(int row, int col, unsigned long n, int width, unsigned char attr);
```

**概要:** 指定された行・列に符号なし整数を 10 進数で右揃え表示する。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `row` | `int` | 表示行番号 (0-24) |
| `col` | `int` | 表示開始列番号 (0-79) |
| `n` | `unsigned long` | 出力する数値 |
| `width` | `int` | 表示幅 (右揃え、余白はスペースで埋める) |
| `attr` | `unsigned char` | 文字属性バイト |

**戻り値:** なし

**処理内容:**

1. 数値を 10 進数文字列に変換する (`buf[12]` にリトルエンディアンで格納後、反転)
2. `width` から数字の桁数を引いた分をスペースで左パディングする
3. VGA メモリに直接書き込む

**呼び出し元:** ユーザータスク (カウンタ値のステータス表示など)

**注意点:** `vga_write_at` と同様、`scrolltop` を考慮しない直接書き込み。最大 11 桁の数値を表示可能 (バッファサイズの制約)。

## video.s について

`video.s` は `video.c` を GCC でコンパイルして生成されたアセンブリ出力である。手書きのアセンブリではなく、GCC のコンパイル結果がそのままソースツリーに含まれている形態である。ファイル先頭のコメント:

```asm
.file  "video.c"
.version  "01.01"
gcc2_compiled.:
```

これは GCC 2.x 系のコンパイラが生成したことを示している。現在のビルドでは `video.c` が直接コンパイルされるため、`video.s` は参照されない。

## 補足

### VGA テキストモードのメモリレイアウト

VGA テキストモードでは、1 文字が 2 バイト (ワード) で表現される:

```
アドレス   内容
0xB8000    文字コード (文字 0,0)
0xB8001    属性バイト (文字 0,0)
0xB8002    文字コード (文字 0,1)
0xB8003    属性バイト (文字 0,1)
...
0xB80A0    文字コード (文字 1,0)  (= 0xB8000 + 160)
```

属性バイトの構成:

```
ビット 7:    点滅 (B)
ビット 6-4:  背景色 (RGB)
ビット 3:    高輝度 (I)
ビット 2-0:  前景色 (RGB)
```

`G_ATTR = 0x02` は「黒背景 + 緑前景」を意味する。

### スクロールの仕組み

このドライバではハードウェアスクロールを使用している:

1. 6845 の表示開始アドレス (`G_VID_ORG`) を 80 文字分ずらすことで、VGA メモリの読み出し開始位置を変更する
2. VGA メモリは `G_HEIGHT` (26 行) 分使用される
3. `scrolltop` が `80 * 25` (2000) を超えた場合、VGA メモリの後方から先頭にデータをコピーしてラップアラウンドする
4. ソフトウェアスクロール (全画面コピー) と比較してハードウェアスクロールは高速だが、ラップアラウンド処理が必要になる

### スピンロックによる排他制御

`printk` と `printk2` は `smp_lock(&video_lk)` / `smp_unlock(&video_lk)` でスピンロックを取得する。これにより、SMP 環境で CPU 0 と CPU 1 が同時に画面出力を行っても、文字化けやカーソル位置の不整合が発生しない。

### printk の使用制限

`printk` はカーネルモード (Ring 0) 専用である。ユーザータスクからの画面出力には以下の代替手段を使用する:

- システムコール経由の `printk2` (ISR コンテキスト)
- `vga_write_at` / `vga_write_dec_at` (直接 VGA 書き込み、Ring 3 安全)
