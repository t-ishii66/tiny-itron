# include/config.h - カーネル構成定数

## 概要

tiny-itron カーネルの構成定数を定義するヘッダファイル。
優先度範囲、カーネル識別情報、キューイング上限、ビットパターン幅、タイマーティック、各種オブジェクトの最大 ID を規定する。

## 優先度範囲

### タスク優先度

| 定数 | 値 | 説明 |
|------|-----|------|
| `TMIN_TPRI` | `1` | タスク最小優先度 (最高優先) |
| `TMAX_TPRI` | `16` | タスク最大優先度 (最低優先) |

優先度は数値が小さいほど高優先度。本カーネルでは 1 (最高) から 16 (最低) まで使用可能。

### メッセージ優先度

| 定数 | 値 | 説明 |
|------|-----|------|
| `TMIN_MPRI` | `1` | メッセージ最小優先度 |
| `TMAX_MPRI` | `16` | メッセージ最大優先度 |

メールボックスの `TA_MPRI` 属性使用時にメッセージの優先度として使用される。

## カーネル識別情報

| 定数 | 値 | 説明 |
|------|--------|------|
| `TKERNEL_MAKER` | `0x0000` | メーカーコード (実験システム) |
| `TKERNEL_PRID` | `0x0001` | カーネル製品 ID |
| `TKERNEL_SPVER` | `0x5400` | ITRON 仕様バージョン (v4.0.0) |
| `TKERNEL_PRVER` | `0x0000` | カーネルバージョン (0.0.0) |

`TKERNEL_SPVER` の値 `0x5400` は Micro ITRON 仕様バージョン 4.0.0 を示す。
上位バイト `0x54` は仕様コード、下位バイト `0x00` はマイナーバージョンを表す。

`ref_ver` サービスコールで `T_RVER` 構造体を通じてこれらの値が返される。

## キューイング・ネスト上限

| 定数 | 値 | 説明 |
|------|-----|------|
| `TMAX_ACTCNT` | `16` | タスク起動要求キューイング上限 |
| `TMAX_WUPCNT` | `16` | タスク起床要求キューイング上限 |
| `TMAX_SUSCNT` | `16` | タスク強制待ちネスト上限 |

`act_tsk` の連続呼び出しでキューイングできる起動要求数、`wup_tsk` の起床要求数、`sus_tsk` のネスト数をそれぞれ制限する。上限を超えると `E_QOVR` が返される。

## ビットパターン幅

| 定数 | 値 | 説明 |
|------|-----|------|
| `TBIT_TEXPTN` | `32` | タスク例外要因ビットパターン幅 |
| `TBIT_FLGPTN` | `32` | イベントフラグビットパターン幅 |
| `TBIT_RDVPTN` | `32` | ランデブビットパターン幅 |

全て 32 ビット。i386 の `UINT` (32 ビット) に対応する。

## タイマーティック

| 定数 | 値 | 説明 |
|------|------|------|
| `TIC_NUME` | `17` | タイマーティック分子 (~17 ミリ秒) |
| `TIC_DENO` | `1000` | タイマーティック分母 |

タイマーティック = `TIC_NUME / TIC_DENO` = 17/1000 秒 ≈ 17 ミリ秒。
PIT を HZ=60 (カウンタ = 1193182/60 = 19886) に設定しているため、実際の周期は
1193182/19886 ≈ 16.7ms。`TIC_NUME=17` は最も近い整数値。

`isig_tim` がタイマー割り込みごとに呼ばれ、システム時刻を 17ms ずつ進める。

## セマフォ上限

| 定数 | 値 | 説明 |
|------|-------|------|
| `TMAX_MAXSEM` | `65535` | セマフォ最大資源数の上限 |

`T_CSEM.maxsem` に指定できる最大値。

## オブジェクト最大 ID

ITRON 仕様外の独自定数。各種カーネルオブジェクトの最大 ID 数を規定する。
ID は 1 から始まるため、使用可能な ID は 1 から MAX_xxxID まで。

### 主要オブジェクト

| 定数 | 値 | 説明 |
|------|-----|------|
| `MAX_TSKID` | `16` | タスク最大 ID |
| `MAX_SEMID` | `16` | セマフォ最大 ID |
| `MAX_FLGID` | `16` | イベントフラグ最大 ID |
| `MAX_DTQID` | `16` | データキュー最大 ID |
| `MAX_MBXID` | `16` | メールボックス最大 ID |
| `MAX_MTXID` | `16` | ミューテックス最大 ID |

### メモリ管理

| 定数 | 値 | 説明 |
|------|-----|------|
| `MAX_MPFID` | `16` | 固定長メモリプール最大 ID |
| `MAX_MPLID` | `16` | 可変長メモリプール最大 ID |

### メッセージ・ランデブ

| 定数 | 値 | 説明 |
|------|-----|------|
| `MAX_MBFID` | `16` | メッセージバッファ最大 ID |
| `MAX_PORID` | `16` | ランデブポート最大 ID |

### 時間管理

| 定数 | 値 | 説明 |
|------|-----|------|
| `MAX_CYCID` | `16` | 周期ハンドラ最大 ID |
| `MAX_ALMID` | `16` | アラームハンドラ最大 ID |

### 割り込み・システム

| 定数 | 値 | 説明 |
|------|-----|------|
| `MAX_INHID` | `16` | 割り込みハンドラ最大 ID |
| `MAX_EXCID` | `16` | CPU 例外ハンドラ最大 ID |
| `MAX_SVCID` | `16` | 拡張サービスコール最大 ID |
| `MAX_ISRID` | `15` | 割り込みサービスルーチン最大 ID (0-15) |
| `MAX_FNCD` | `16` | ファンクションコード最大数 |
| `MAX_EXC` | `15` | CPU 例外最大数 (0-15) |

`MAX_ISRID` と `MAX_EXC` は 0 から始まるため、0-15 の 16 個が使用可能。
