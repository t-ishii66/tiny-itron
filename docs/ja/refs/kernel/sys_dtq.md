# sys_dtq -- データキュー管理システムコール

対象ファイル: `kernel/sys_dtq.c`, `kernel/sys_dtq.h`

## 概要

Micro ITRON v4.0.0 仕様に基づくデータキュー管理システムコールの実装。データキューの生成・削除・送信・受信・ポーリング・タイムアウト付き操作・強制送信・参照の操作を提供する。

データキューはリングバッファ方式で実装されており、読み出しポインタ (`r`) と書き込みポインタ (`w`) で管理される。各エントリは `VW` (32 ビットポインタ) 型のデータを 1 つ格納する。送信待ちキュー (`wlink_w`) と受信待ちキュー (`wlink_r`) の 2 つの待ちキューを持つ。

全ての公開関数は第1引数に `W apic` (APIC ID、0 または 1 に正規化済み) を取る。

## 定数・マクロ

データキュー属性 (`include/itron.h`):

| 定数 | 値 | 意味 |
|------|-----|------|
| `TA_TFIFO` | 0x00 | 待ちキューを FIFO 順で管理 |
| `TA_TPRI` | 0x01 | 待ちキューをタスク優先度順で管理 |

設定上限 (`include/config.h`):

| 定数 | 値 |
|------|-----|
| `MAX_DTQID` | 16 |

## グローバル変数

| 変数 | 型 | 定義場所 | 説明 |
|------|-----|----------|------|
| `dtq[]` | `T_DTQ[MAX_DTQID]` | `kernel/val.h` (extern) | データキュー管理ブロック配列 |

### T_DTQ 構造体 (kernel/types.h)

```c
typedef struct t_dtq {
    T_LINK      wlink_w;        /* 送信待ちキュー */
    T_LINK      wlink_r;        /* 受信待ちキュー */
    ATR         dtqatr;         /* データキュー属性 */
    UINT        dtqcnt;         /* データキュー容量 (エントリ数) */
    VP          dtq;            /* リングバッファ先頭アドレス */
    STAT        act;            /* 活性状態 (0: 未生成, 1: 生成済み) */
    STAT        dtq_alloc;      /* カーネルによるメモリ割り当て (1: カーネル割り当て) */
    VW          r;              /* 読み出しポインタ (リングバッファ) */
    VW          w;              /* 書き込みポインタ (リングバッファ) */
} T_DTQ;
```

### T_TSK 内のデータキュー関連フィールド

```c
VP_INT      data;       /* 送信待ち時に保持するデータ */
VP_INT*     p_data;     /* 受信待ち時のデータ格納先ポインタ */
```

### リングバッファの構造

```
dtq (先頭)                                     dtq + dtqcnt
    |                                               |
    v                                               v
    +---+---+---+---+---+---+---+---+
    |   |   | D | D | D |   |   |   |
    +---+---+---+---+---+---+---+---+
              ^           ^
              |           |
              r           w
```

- `r == w` の場合: バッファは空
- `(w + 1) % dtqcnt == r` の場合: バッファは満杯
- 各エントリは `VW` (unsigned long ポインタ) 型 (4 バイト)
- ポインタは `VW` 型 (バイトアドレスではなくワードポインタ) として管理される

## 関数リファレンス

### dtq_init

```c
void dtq_init(void)
```

**概要:** 全データキュー管理ブロックを初期化する。

**引数:** なし

**戻り値:** なし

**処理内容:**
1. `dtq[1]` から `dtq[MAX_DTQID-1]` まで反復
2. 送信待ちキュー (`wlink_w`) と受信待ちキュー (`wlink_r`) を自己参照 (空リスト) に初期化
3. `act` を 0 (未生成) に設定

**呼び出し元:** `itron_init()` (カーネル初期化時)

**注意点:** ループはインデックス 1 から開始 (`i = 1`)。インデックス 0 は初期化されない。

### sys_cre_dtq

```c
ER sys_cre_dtq(W apic, ID dtqid, T_CDTQ* pk_cdtq)
```

**概要:** データキューを生成する。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `dtqid` | `ID` | データキュー ID (1〜MAX_DTQID) |
| `pk_cdtq` | `T_CDTQ*` | データキュー生成情報パケット |

T_CDTQ 構造体:

```c
typedef struct t_cdtq {
    ATR     dtqatr;     /* データキュー属性 */
    UINT    dtqcnt;     /* データ容量 */
    VP      dtq;        /* バッファ先頭アドレス (NULL: カーネルが割り当て) */
} T_CDTQ;
```

**戻り値:**

| 値 | 意味 |
|-----|------|
| `E_OK` | 正常終了 |
| `E_ID` | 不正な ID (範囲外) |
| `E_OBJ` | 既に生成済み |

**処理内容:**
1. ID 範囲チェック
2. `act != 0` なら `E_OBJ`
3. `act = 1` に設定
4. `dtqatr`, `dtqcnt` を設定
5. バッファ割り当て:
   - `pk_cdtq->dtq` が NULL → `kmem_alloc(sizeof(VW) * dtqcnt)` でカーネル割り当て (Supervisor ページ)、`dtq_alloc = 1`
   - NULL でない → ユーザー提供バッファを使用、`dtq_alloc = 0`
6. `r` と `w` を共にバッファ先頭にキャスト (`(VW)dtq[dtqid].dtq`)

**呼び出し元:** ユーザーコード (`cre_dtq`)

**注意点:** `r` と `w` は `VW` 型 (`unsigned long*`) にキャストされる。ポインタ演算は `VW` の単位 (4 バイト) で行われる。

### sys_acre_dtq

```c
ER_ID sys_acre_dtq(W apic, T_CDTQ* pk_cdtq)
```

**概要:** 空きデータキュー ID を自動割り当てしてデータキューを生成する。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `pk_cdtq` | `T_CDTQ*` | データキュー生成情報パケット |

**戻り値:** `E_OK`

**処理内容:**
1. `dtq[1]` から `dtq[MAX_DTQID]` まで走査し、`act == 0` の未使用エントリを検索
2. 見つかれば `sys_cre_dtq(apic, i, pk_cdtq)` に委譲して生成
3. 成功時は割り当てた ID を返す
4. 空きが見つからなければエラーを返す

### sys_del_dtq

```c
ER sys_del_dtq(W apic, ID dtqid)
```

**概要:** データキューを削除する。待ちタスクがあれば全て `E_DLT` で起床する。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `dtqid` | `ID` | データキュー ID |

**戻り値:** `E_OK`

**処理内容:**
1. ID 範囲チェック、存在チェック
2. 受信待ちキュー (`wlink_r`) を走査:
   - 各待ちタスクの `tlink` を `sched_timeout_rem_if_exist()` でタイムアウトキューから除去
   - タスクを `TTS_RDY` に変更
   - `proc_set_return_value(t->proc, E_DLT)` で戻り値を設定
   - `sched_ins()` でレディキューに挿入
3. 送信待ちキュー (`wlink_w`) を走査: 同様に全待ちタスクを起床
4. 両待ちキューを空にリセット
5. `dtq_alloc == 1` なら `kmem_free()` でバッファ解放
6. `act = 0` に設定

**呼び出し元:** ユーザーコード (`del_dtq`)

### sys_psnd_dtq

```c
ER sys_psnd_dtq(W apic, ID dtqid, VP_INT data)
```

**概要:** データキューにポーリング (非ブロッキング) 送信する。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `dtqid` | `ID` | データキュー ID |
| `data` | `VP_INT` | 送信データ |

**戻り値:**

| 値 | 意味 |
|-----|------|
| `E_OK` | 正常終了 (送信成功) |
| `E_TMOUT` | バッファ満杯 |

**処理内容:**
1. 次の書き込み位置を計算 (`next_w = w + 1`、末尾なら先頭に巻き戻し)
2. 受信待ちタスクが存在する場合:
   - `sched_timeout_rem_if_exist()` でタイムアウトキューから tlink を除去
   - `wlink_rem()` で待ちキューから削除
   - タスクを `TTS_RDY` に変更
   - タスクの `p_data` にデータを書き込み
   - `sched_ins()` でレディキューに挿入
3. 受信待ちタスクがいない場合:
   - `next_w == r` (バッファ満杯) なら `E_TMOUT`
   - 空きがあればバッファに書き込み、`w` を進める

**呼び出し元:** ユーザーコード (`psnd_dtq`)

### ipsnd_dtq

```c
ER ipsnd_dtq(W apic, ID dtqid, VP_INT data)
```

**概要:** 割り込みハンドラからデータキューに送信する。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `dtqid` | `ID` | データキュー ID |
| `data` | `VP_INT` | 送信データ |

**戻り値:**

| 値 | 意味 |
|-----|------|
| `E_OK` | 正常終了 |
| `E_TMOUT` | バッファ満杯 |

**処理内容:**
1. 次の書き込み位置を計算 (巻き戻し含む)
2. 受信待ちタスクが存在する場合:
   - `sched_timeout_rem_if_exist()` でタイムアウトキューから tlink を除去
   - `wlink_rem()` で待ちキューから削除
   - タスクを `TTS_RDY` に変更
   - タスクの `p_data` にデータを書き込み
   - `sched_ins()` でレディキューに挿入

3. バッファ満杯なら `E_TMOUT`
4. 空きがあればバッファに書き込み

**呼び出し元:** キーボード割り込みハンドラ等

**注意点:** `sys_psnd_dtq` との主な違いは、受信待ちタスクを起床した後に `sched_next_tsk()` を呼び出す点。これにより割り込みコンテキストから即座にタスク切り替えが発生する。

### sys_tsnd_dtq

```c
ER sys_tsnd_dtq(W apic, ID dtqid, VP_INT data, TMO tmout)
```

**概要:** タイムアウト付きでデータキューに送信する。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `dtqid` | `ID` | データキュー ID |
| `data` | `VP_INT` | 送信データ |
| `tmout` | `TMO` | タイムアウト値 |

**戻り値:**

| 値 | 意味 |
|-----|------|
| `E_OK` | 正常終了 |
| `E_TMOUT` | ポーリング失敗 (TMO_POL 時) |

**処理内容:**
1. 次の書き込み位置を計算
2. 受信待ちタスクが存在する場合:
   - `sched_timeout_rem_if_exist()` で tlink を除去し、直接データを渡して起床 (バッファを経由しない)
3. バッファ満杯の場合:
   - `TMO_POL` なら `E_TMOUT`
   - 待ちキュー (`wlink_w`) にタスクを挿入:
     - `TA_TFIFO` → `ins_fifo()` で FIFO 順
     - それ以外 → `ins_pri()` で優先度順
   - `tsk[].data` に送信データを保存
   - `TMO_FEVR` → `sys_slp_tsk()` で永久待ち
   - タイムアウト指定 → `sys_tslp_tsk()`
4. バッファに空きがある場合: バッファに書き込み、`w` を進める

**呼び出し元:** `sys_snd_dtq`、ユーザーコード (`tsnd_dtq`)

**注意点:** 送信待ちに入る際、データは `tsk[].data` に保存される。バッファに空きが出来た際に、このデータが取り出されてバッファに書き込まれる処理は受信側で行われる想定。

### sys_fsnd_dtq

```c
ER sys_fsnd_dtq(W apic, ID dtqid, VP_INT data)
```

**概要:** データキューに強制送信する。バッファが満杯の場合、最古のデータを破棄して書き込む。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `dtqid` | `ID` | データキュー ID |
| `data` | `VP_INT` | 送信データ |

**戻り値:** `E_OK`

**処理内容:**
1. 次の書き込み位置を計算
2. 受信待ちタスクが存在する場合: `sched_timeout_rem_if_exist()` で tlink を除去し、直接データを渡して起床
3. バッファ満杯の場合:
   - `r` を 1 つ進める (最古のデータを破棄、巻き戻し処理あり)
4. バッファに書き込み、`w` を進める

**呼び出し元:** ユーザーコード (`fsnd_dtq`)

**注意点:** 強制送信ではブロックせず、必ず成功する。バッファ満杯時は読み出しポインタ `r` を進めて最古のデータを上書きする。

### sys_trcv_dtq

```c
ER sys_trcv_dtq(W apic, ID dtqid, VP_INT* p_data, TMO tmout)
```

**概要:** タイムアウト付きでデータキューから受信する。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `dtqid` | `ID` | データキュー ID |
| `p_data` | `VP_INT*` | 受信データの格納先 |
| `tmout` | `TMO` | タイムアウト値 |

**戻り値:**

| 値 | 意味 |
|-----|------|
| `E_OK` | 正常終了 |
| `E_TMOUT` | ポーリング失敗 (TMO_POL 時) |

**処理内容:**
1. `w != r` (データあり) の場合:
   - バッファからデータを読み出し、`r` を進める
2. データなしの場合:
   - `TMO_POL` なら `E_TMOUT`
   - 受信待ちキュー (`wlink_r`) にタスクを挿入:
     - `TA_TFIFO` → `ins_fifo()` で FIFO 順
     - それ以外 → `ins_pri()` で優先度順
   - `tsk[].p_data = p_data` に受信先ポインタを保存
   - `TMO_FEVR` → `sys_slp_tsk()` で永久待ち
   - タイムアウト指定 → `sys_tslp_tsk()`

**呼び出し元:** `sys_rcv_dtq`、ユーザーコード (`trcv_dtq`)

**注意点:**
- 送信待ちタスクからのデータ受け取り処理 (送信待ちキューのチェック) はコメントアウトされている (`#if 0` ブロック)。現在の実装ではバッファが空で送信待ちタスクがいる場合、送信待ちタスクのデータは受け取れない。

### sys_ref_dtq

```c
ER sys_ref_dtq(W apic, ID dtqid, T_RDTQ* pk_rdtq)
```

**概要:** データキューの状態を参照する。

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `apic` | `W` | APIC ID |
| `dtqid` | `ID` | データキュー ID |
| `pk_rdtq` | `T_RDTQ*` | 参照情報の格納先 |

T_RDTQ 構造体:

```c
typedef struct t_rdtq {
    ID      stskid;     /* 送信待ちキュー先頭のタスク ID */
    ID      rtskid;     /* 受信待ちキュー先頭のタスク ID */
    UINT    sdtqcnt;    /* バッファ内のデータ数 */
} T_RDTQ;
```

**戻り値:** `E_OK`、`E_ID`、`E_NOEXS`

**処理内容:**
1. ID 範囲チェック、存在チェック
2. 送信待ちキューが空なら `stskid = TSK_NONE`、空でなければ先頭タスクの ID を設定
3. 受信待ちキューが空なら `rtskid = TSK_NONE`、空でなければ先頭タスクの ID を設定
4. `sdtqcnt` にバッファ内のデータ数を設定

## 補足

### リングバッファの管理方式

リングバッファのポインタ `r` と `w` は `VW` 型 (`unsigned long*`) であり、バイトアドレスではなくワードポインタとして管理される。ポインタの巻き戻し判定は以下のように行われる:

```c
if (next_w >= (VW)dtq[dtqid].dtq + dtq[dtqid].dtqcnt) {
    next_w = (VW)dtq[dtqid].dtq;
}
```

`VW` 型へのキャストと演算により、`dtqcnt` 個分のワード単位で循環する。

### バッファ空判定と満杯判定

- **空**: `w == r`
- **満杯**: `(w + 1) を巻き戻した値 == r` (1 エントリ分を番兵として使うため、実際の格納可能数は `dtqcnt - 1`)

### 送信待ちタスクと受信待ちタスクの直接データ受け渡し

受信待ちタスクが存在する状態で送信が行われた場合、バッファを経由せず直接タスクの `p_data` ポインタにデータが書き込まれる。これにより不必要なコピーとバッファ操作を回避している。

### デモでの使用例

キーボード入力のデモで使用される。キーボード割り込みハンドラ (`key_intr`) が `ipsnd_dtq()` でキーコードをデータキューに送信し、`kbd_task` (Task 4) が `rcv_dtq()` でデータを受信して VGA に表示する。

### 制限事項

- `sys_trcv_dtq`: 送信待ちタスクからのデータ受け取りがコメントアウトされている (`#if 0` ブロック)
- 排他制御は Big Kernel Lock (`kernel_lk`) により呼び出し元で保護されている (セマフォ・イベントフラグと同様)
