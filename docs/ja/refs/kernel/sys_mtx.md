# sys_mtx.c / sys_mtx.h

対象ファイル: `kernel/sys_mtx.c`, `kernel/sys_mtx.h`

## 概要

ミューテックス (mutex) のシステムコール実装。Micro ITRON v4.0.0 仕様に基づく、優先度継承・優先度上限プロトコル付きの排他制御機構を提供する。

ミューテックスはバイナリロックであり、1つのタスクのみがロックを保持できる。ロック保持中に高優先度のタスクがロック待ちに入ると、優先度継承プロトコルにより保持タスクの実行優先度が引き上げられる。ロック解放時に他にミューテックスを保持していなければ、基本優先度 (`tskbpri`) に復帰する。

### データ構造

```c
typedef struct t_mtx {
    ID        tskid;      /* ロック保持タスク ID */
    T_LINK    wlink;      /* ロック待ちタスクキュー */
    ATR       mtxatr;     /* ミューテックス属性 (TA_TFIFO/TA_TPRI, TA_INHERIT/TA_CEILING) */
    PRI       ceilpri;    /* 上限優先度 (TA_CEILING 時) */
    W         mtxlock;    /* ロック状態 (0=解放, 1=ロック中) */
    STAT      act;        /* 生成済みフラグ */
} T_MTX;
```

タスク構造体の関連フィールド:
- `tsk[].mtxcnt`: 保持中のミューテックス数
- `tsk[].tskbpri`: 基本優先度 (ミューテックス取得前の優先度)
- `tsk[].tskpri`: 現在の実行優先度 (優先度継承で変化する)

### 定数

- `MAX_MTXID`: 16 (`include/config.h`)

## 関数リファレンス

### mtx_init

```c
void mtx_init(void)
```

**概要:** 全ミューテックスを初期化する。

**引数:** なし

**戻り値:** なし

**処理内容:**
1. ID 1 から `MAX_MTXID - 1` まで全エントリをループ (ループ条件は `i < MAX_MTXID`)
2. 各エントリの待ちキュー (`wlink`) を空の双方向リンクリストに初期化
3. `act` フラグを 0 (未生成) に設定

### sys_cre_mtx

```c
ER sys_cre_mtx(W apic, ID mtxid, T_CMTX* pk_cmtx)
```

**概要:** 指定 ID でミューテックスを生成する。

**引数:**
| 名前 | 型 | 説明 |
|------|------|------|
| apic | W | APIC ID |
| mtxid | ID | ミューテックス ID (1 ~ MAX_MTXID) |
| pk_cmtx | T_CMTX* | 生成情報パケット (mtxatr, ceilpri) |

**戻り値:**
| 値 | 説明 |
|------|------|
| E_OK | 正常終了 |
| E_ID | ID が範囲外 |
| E_OBJ | 既に生成済み |

**処理内容:**
1. ID 範囲チェック
2. 既存オブジェクトチェック
3. `pk_cmtx` から `mtxatr`, `ceilpri` をコピー
4. `act = 1` で生成完了

**注意:** 明示的な `return E_OK` が欠落している (コンパイラ依存の戻り値)。

### sys_del_mtx

```c
ER sys_del_mtx(W apic, ID mtxid)
```

**概要:** ミューテックスを削除する。待ちタスクには E_DLT を通知する。

**引数:**
| 名前 | 型 | 説明 |
|------|------|------|
| apic | W | APIC ID |
| mtxid | ID | ミューテックス ID |

**戻り値:**
| 値 | 説明 |
|------|------|
| E_OK | 正常終了 |
| E_ID | ID が範囲外 |
| E_NOEXS | オブジェクト未生成 |

**処理内容:**
1. ID・存在チェック
2. 待ちキューの全タスクをループ:
   - タスク状態を `TTS_RDY` に変更
   - 戻り値レジスタ (`proc->reg[EAX]`) に `E_DLT` を設定
   - スケジューラキューに挿入
3. `act = 0` で削除完了

**注意:** ロック保持タスクの優先度復元処理はコメントとして記載されているが、実装されていない。

### sys_tloc_mtx

```c
ER sys_tloc_mtx(W apic, ID mtxid, TMO tmout)
```

**概要:** ミューテックスをタイムアウト付きでロックする。優先度継承プロトコルを実行する。

**引数:**
| 名前 | 型 | 説明 |
|------|------|------|
| apic | W | APIC ID |
| mtxid | ID | ミューテックス ID |
| tmout | TMO | タイムアウト (TMO_POL/TMO_FEVR/正値) |

**戻り値:**
| 値 | 説明 |
|------|------|
| E_OK | 正常終了 (ロック取得成功) |
| E_ID | ID が範囲外 |
| E_NOEXS | オブジェクト未生成 |
| E_ILUSE | 自タスクが既にロック保持 / 上限優先度違反 |
| E_TMOUT | タイムアウト (TMO_POL 時) |

**処理内容:**
1. ID・存在チェック
2. ミューテックスが未ロック (`mtxlock == 0`) の場合:
   - `mtxlock = 1` に設定
   - 呼び出しタスクの `mtxcnt` をインクリメント
   - E_OK を返す
3. 自タスクが既にロック保持中なら E_ILUSE を返す
4. `TA_CEILING` 属性の場合、呼び出しタスクの基本優先度が上限優先度 (`ceilpri`) より低い (数値が大きい) なら E_ILUSE を返す
5. `TMO_POL` なら E_TMOUT を返す
6. 待ちキューに挿入 (`TA_TFIFO` で FIFO 順、それ以外は優先度順)
7. **優先度継承**: ロック保持タスクの優先度が待ちタスクより低い (数値が大きい) 場合:
   - 保持タスクの `tskpri` を待ちタスクの優先度に引き上げ
   - 保持タスクをスケジューラキューから除去し、新しい優先度で再挿入
8. `TMO_FEVR` なら `sys_slp_tsk` でブロック、正値なら `sys_tslp_tsk` でタイムアウト付きブロック

**注意:** 優先度継承の判定条件は `holder->tskpri > waiter->tskpri` (ITRON では数値が小さい方が高優先度なので、数値が大きい=低優先度の保持タスクを引き上げる)。

### sys_unl_mtx

```c
ER sys_unl_mtx(W apic, ID mtxid)
```

**概要:** ミューテックスのロックを解放する。待ちタスクがあればロック権を移譲する。

**引数:**
| 名前 | 型 | 説明 |
|------|------|------|
| apic | W | APIC ID |
| mtxid | ID | ミューテックス ID |

**戻り値:**
| 値 | 説明 |
|------|------|
| E_OK | 正常終了 |
| E_ID | ID が範囲外 |
| E_NOEXS | オブジェクト未生成 |
| E_ILUSE | ロックされていない |

**処理内容:**
1. ID・存在チェック
2. `mtxlock == 0` (未ロック) なら E_ILUSE を返す
3. 待ちキューに待ちタスクがある場合:
   - 先頭タスクにロック権を移譲 (`mtxlock = 1`, `tskid` を更新)
   - 待ちタスクの `mtxcnt` をインクリメント
   - 待ちキューから除去
4. 解放タスクの `mtxcnt` をデクリメント
5. `mtxcnt` が 0 (他にミューテックスを保持していない) の場合:
   - 基本優先度 (`tskbpri`) と現在優先度 (`tskpri`) が異なれば、基本優先度に復帰
   - スケジューラキューから除去し、基本優先度で再挿入

## 補足

- 優先度継承プロトコル: ミューテックスロック待ちに入るタスクの優先度が、ロック保持タスクの優先度より高い場合、保持タスクの実行優先度を一時的に引き上げる。これにより優先度逆転問題を軽減する。
- `TA_CEILING` (優先度上限プロトコル): 生成時に指定した上限優先度 (`ceilpri`) より低い優先度のタスクのみがロック可能。上限優先度を超えるタスクが `tloc_mtx` を呼ぶと E_ILUSE が返る。
- `mtxcnt` による多重ミューテックス管理: タスクが複数のミューテックスを保持している場合、全てを解放するまで基本優先度に復帰しない。
- `sys_ploc_mtx`, `sys_acre_mtx` は未実装。
