# table.s / 386.c / 386.h / 386P.h

対象ファイル: `i386/table.s`, `i386/386.c`, `i386/386.h`, `i386/386P.h`, `i386/genasm.c`

## 概要

GDT (Global Descriptor Table) の定義と、セグメントディスクリプタ・ゲートディスクリプタの操作関数群。

- **table.s**: `genasm.c` によって自動生成されるアセンブリファイル。GDT の初期内容を `.byte` ディレクティブで定義する。32 エントリ x 8 バイト = 256 バイトの GDT データを含み、`.org 4096` で 4KB にパディングされる。物理アドレス 0x2000 (AL_GDT) に配置される。
- **genasm.c**: ホスト側で実行される GDT 生成プログラム。GDT エントリを C 構造体で構築し、table.s をバイナリ形式で出力する。
- **386.c**: カーネル内で GDT/IDT エントリを動的に設定する関数 (`set_gdt`, `set_gate`)。TSS セグメントや IDT ゲートの設定に使用される。
- **386.h**: セグメントディスクリプタ (`seg_t`) とゲートディスクリプタ (`gate_t`) の構造体定義、セグメント/ゲートタイプ定数。
- **386P.h**: プライベートヘッダ。セグメントタイプマクロ (現在は空定義)。

## 構造体・型

### seg_t (セグメントディスクリプタ)

```c
typedef struct seg {
    unsigned short  limit_l;   /* リミット下位 16 ビット (ビット 0-15) */
    unsigned short  base_l;    /* ベースアドレス下位 16 ビット (ビット 0-15) */
    unsigned char   base_m;    /* ベースアドレス中位 8 ビット (ビット 16-23) */
    unsigned char   type;      /* タイプ + DPL + Present (アクセスバイト) */
    unsigned char   limit_h;   /* リミット上位 4 ビット + G + D/B + AVL フラグ */
    unsigned char   base_h;    /* ベースアドレス上位 8 ビット (ビット 24-31) */
} seg_t;
```

**概要:** i386 のセグメントディスクリプタ (8 バイト) を表す構造体。GDT エントリとして使用される。

**フィールド詳細:**

| フィールド | サイズ | 説明 |
|-----------|--------|------|
| `limit_l` | 2 バイト | セグメントリミットのビット 0-15 |
| `base_l` | 2 バイト | セグメントベースアドレスのビット 0-15 |
| `base_m` | 1 バイト | セグメントベースアドレスのビット 16-23 |
| `type` | 1 バイト | アクセスバイト: P(1) + DPL(2) + S(1) + Type(4) |
| `limit_h` | 1 バイト | 上位 4 ビット: G + D/B + L + AVL、下位 4 ビット: リミットのビット 16-19 |
| `base_h` | 1 バイト | セグメントベースアドレスのビット 24-31 |

### gate_t (ゲートディスクリプタ)

```c
typedef struct gate {
    unsigned short  offset_l;  /* オフセット下位 16 ビット (ビット 0-15) */
    unsigned short  sel;       /* セグメントセレクタ */
    unsigned char   count;     /* パラメータカウント (コールゲート用) */
    unsigned char   type;      /* タイプ + DPL + Present */
    unsigned short  offset_h;  /* オフセット上位 16 ビット (ビット 16-31) */
} gate_t;
```

**概要:** i386 のゲートディスクリプタ (8 バイト) を表す構造体。IDT エントリ (割り込みゲート/トラップゲート) やコールゲートとして使用される。

**フィールド詳細:**

| フィールド | サイズ | 説明 |
|-----------|--------|------|
| `offset_l` | 2 バイト | ハンドラのオフセットアドレスのビット 0-15 |
| `sel` | 2 バイト | ハンドラが存在するコードセグメントのセレクタ |
| `count` | 1 バイト | パラメータワードカウント (コールゲートのみ、0-31) |
| `type` | 1 バイト | P(1) + DPL(2) + 0(1) + Type(4) |
| `offset_h` | 2 バイト | ハンドラのオフセットアドレスのビット 16-31 |

## 関数リファレンス

### set_gdt

```c
void set_gdt(short n, unsigned long base, unsigned long limit,
             unsigned char type, unsigned char dpl, unsigned char gda);
```

**概要:** GDT のエントリを初期化する。指定されたバイトオフセット位置のセグメントディスクリプタに、ベースアドレス・リミット・タイプ・DPL・粒度を設定する。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `n` | `short` | GDT 内のバイトオフセット (セレクタ値: 0x08, 0x10, 0x20 など) |
| `base` | `unsigned long` | セグメントベースアドレス (32 ビットリニアアドレス) |
| `limit` | `unsigned long` | セグメントリミット (20 ビット) |
| `type` | `unsigned char` | セグメントタイプ (ST_CODE, ST_DATA, ST_STACK, ST_TSS 等) |
| `dpl` | `unsigned char` | ディスクリプタ特権レベル (0=カーネル, 3=ユーザー) |
| `gda` | `unsigned char` | 粒度 + D/B + AVL フラグ (_16BIT=0x00, _32BIT=0xC0) |

**戻り値:** なし

**処理内容:**

1. バイトオフセット `n` を 8 で割ってエントリインデックスに変換する
2. `base` を 3 つのフィールド (`base_l`, `base_m`, `base_h`) に分割して格納する
3. `limit` を 2 つのフィールド (`limit_l`, `limit_h`) に分割して格納する
4. `limit_h` の上位 4 ビットに `gda` (粒度フラグ) を加算する
5. `type` に `dpl << 5` を OR して、DPL を含むアクセスバイトを構築する

**呼び出し元:** `genasm.c` の `main()` (GDT 初期生成)、カーネル初期化時の TSS セグメント設定

**注意点:** `genasm.c` 内にも同名の関数が存在する。genasm.c はホスト上で実行される生成プログラムであり、386.c はカーネル内で実行されるランタイム版である。

### set_gate

```c
void set_gate(short n, unsigned long offset, unsigned short sel,
              unsigned char type, unsigned char count);
```

**概要:** IDT のエントリ (ゲートディスクリプタ) を初期化する。割り込みゲートやトラップゲート、コールゲートの設定に使用する。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `n` | `short` | IDT/GDT 内のバイトオフセット |
| `offset` | `unsigned long` | ハンドラのオフセットアドレス (32 ビット) |
| `sel` | `unsigned short` | ハンドラが存在するコードセグメントのセレクタ |
| `type` | `unsigned char` | ゲートタイプ (GT_INTR, GT_TRAP, GT_CALL) |
| `count` | `unsigned char` | パラメータワードカウント (コールゲートでのスタックコピーワード数) |

**戻り値:** なし

**処理内容:**

1. バイトオフセット `n` を 8 で割ってエントリインデックスに変換する
2. `gdt` ポインタからの相対位置を `gate_t*` にキャストする (GDT と IDT は連続配置であるため共通ポインタを使用)
3. `offset` を上位・下位 16 ビットに分割して格納する
4. セレクタ、パラメータカウント、タイプを設定する

**呼び出し元:** カーネル初期化時の IDT 設定 (割り込みハンドラ登録)

**注意点:** `n` は GDT のベースアドレス (`gdt` ポインタ) からの相対バイトオフセットとして計算される。IDT は GDT の直後 (AL_IDT = 0x2100) に配置されるため、IDT エントリを設定する場合は `n` に GDT からの相対オフセットを指定する必要がある。

| 0 | 0x00 | - | - | - | - | - | NULL ディスクリプタ (Intel 規約) |
| 1 | 0x08 | 0x3000 | 0xFFFF | ST_CODE | 0 | 16bit | 16 ビットカーネルコード |
| 2 | 0x10 | 0xB8000 | 0xFFFF | ST_DATA | 0 | 16bit | 16 ビット VGA データ |
| 3 | 0x18 | AL_KERNEL16 (0x3000) | 0x0 | ST_STACK | 0 | 16bit | 16 ビットカーネルスタック |
| 4 | 0x20 | 0x0 | 0xFFFFF | ST_CODE | 0 | 32bit | 32 ビットカーネルコード (SEL_K32_C) |

| 6 | 0x30 | 0x0 | 0x0 | ST_STACK | 0 | 32bit | 32 ビットカーネルスタック (SEL_K32_S) |

| 8 | 0x40 | - | - | - | - | - | TSS1 (実行時に set_gdt で設定) |

| 13 | 0x68 | 0x0 | 0x0 | ST_STACK | 3 | 32bit | 32 ビットユーザースタック (SEL_U32_S) |
| 14-31 | 0x70-0xF8 | - | - | - | - | - | 未使用 (0 初期化) |

## genasm.c の関数リファレンス

### dump_gdt

```c
void dump_gdt(void);
```

**概要:** GDT の全 32 エントリをデバッグ表示する。

**処理内容:** 各エントリのベースアドレス、リミット、タイプを `printf` で出力する。

**呼び出し元:** デバッグ時に `main()` から呼び出し可能 (通常は未使用)

### gen_asm

```c
void gen_asm(void);
```

**概要:** `gdt` 配列の内容を table.s ファイルとして書き出す。

**処理内容:**

1. `A_ASM` ("table.s") ファイルを書き込みモードで開く
2. ヘッダコメントと `.code16`, `.text`, `.globl start` ディレクティブを出力する
3. GDT の 256 バイト (32 エントリ x 8 バイト) を `.byte` ディレクティブで 8 バイトずつ出力する
4. `.org 4096` で 4KB にパディングする

**呼び出し元:** genasm.c の `main()`

### main (genasm.c)

```c
int main(void);
```

**概要:** GDT 生成プログラムのエントリポイント。

**処理内容:**

1. `set_gdt` で各セグメントディスクリプタを設定する:
   - 0x08: 16 ビットカーネルコード (ベース 0x3000)
   - 0x10: 16 ビット VGA データ (ベース 0xB8000)
   - 0x18: 16 ビットカーネルスタック
   - 0x20: 32 ビットカーネルコード (フラットモデル)
   - 0x28: 32 ビットカーネルデータ (フラットモデル)
   - 0x30: 32 ビットカーネルスタック
   - 0x58: 32 ビットユーザーコード (DPL=3)
   - 0x60: 32 ビットユーザーデータ (DPL=3)
   - 0x68: 32 ビットユーザースタック (DPL=3)
2. `gen_asm()` で table.s を生成する

**注意点:** TSS エントリ (0x38, 0x40) とコールゲート (0x70) は genasm.c では設定されず、カーネル実行時に 386.c の `set_gdt` / `set_gate` で動的に設定される。GDT スロット 0x48, 0x50 は未使用。

## 補足

### table.s のバイナリエンコーディング例

table.s のエントリ 4 (セレクタ 0x20: 32 ビットカーネルコード) のバイナリ:

```
.byte 0xff, 0xff, 0x0, 0x0, 0x0, 0x9a, 0xcf, 0x0
```

| バイト | 値 | フィールド | 意味 |
|--------|------|----------|------|
| 0-1 | 0xFFFF | limit_l | リミット下位 = 0xFFFF |
| 2-3 | 0x0000 | base_l | ベース下位 = 0x0000 |
| 4 | 0x00 | base_m | ベース中位 = 0x00 |
| 5 | 0x9A | type | P=1, DPL=0, S=1, Type=コード実行可読 |
| 6 | 0xCF | limit_h | G=1, D=1, リミット上位 = 0xF → リミット = 0xFFFFF x 4KB = 4GB |
| 7 | 0x00 | base_h | ベース上位 = 0x00 → ベース = 0x00000000 |

### 特権レベルとセレクタ

ユーザーモード用セレクタの RPL (Requested Privilege Level) は実際のセレクタ値にエンコードされる:

- `SEL_U32_C` = 0x58 → バイナリ 01011_0_00 → RPL=0 (GDT インデックス 11)。ユーザーモードで使用する際は RPL=3 を OR して 0x5B とする
- `SEL_U32_D` = 0x60 → ユーザーモードでは 0x63
- `SEL_U32_S` = 0x68 → ユーザーモードでは 0x6B
