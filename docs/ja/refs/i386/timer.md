# timer.md

対象ファイル: `i386/timer.c`, `i386/timer.h`, `i386/timerP.h`

## 概要

Intel 8253/8254 PIT (Programmable Interval Timer) のドライバ。CPU 0 のシステムタイマーとして使用され、約 60Hz (16.67ms 周期) で IRQ0 割り込みを発生させる。タイマーティックの計数および VGA 画面への表示を行い、スケジューラのタイムアウト処理を呼び出す。

CPU 1 は PIT を使用せず、Local APIC タイマーのみで動作する。

## 定数・マクロ

### timerP.h

| 定数 | 値 | 説明 |
|------|------|------|
| `FREQ` | `1193182L` | PIT の入力クロック周波数 (Hz)。IBM PC 標準の 1.193182 MHz |
| `SQUARE` | `0x36` | PIT 制御レジスタ設定値。カウンタ 0、LSB/MSB 順、モード 3 (方形波)、バイナリカウント |
| `HZ` | `60` | タイマー割り込み周波数 (Hz) |

### 導出値

| 計算式 | 値 | 説明 |
|--------|------|------|
| `FREQ / HZ` | 19886 (0x4DBE) | PIT カウンタ初期値。実効周期は約 16.67ms |

### io.h (I/O ポートアドレス)

| 定数 | 値 | 説明 |
|------|------|------|
| `IO_TIMER_0` | `0x40` | PIT カウンタ 0 データポート |
| `IO_TIMER_1` | `0x41` | PIT カウンタ 1 データポート |
| `IO_TIMER_2` | `0x42` | PIT カウンタ 2 データポート |
| `IO_TIMER_C` | `0x43` | PIT 制御レジスタ (モードコマンドレジスタ) |

なし。

## 関数リファレンス

### timer_init

```c
void timer_init(void);
```

**概要:** PIT カウンタ 0 を方形波モードで初期化し、周期的な IRQ0 割り込みを設定する。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. カウント値を計算: `count = FREQ / HZ` = 1193182 / 60 = 19886
2. 制御レジスタ (`IO_TIMER_C` = 0x43) に `SQUARE` (`0x36`) を出力
   - カウンタ 0 選択、LSB→MSB 順の 16 ビットアクセス、モード 3 (方形波生成)
3. カウンタ 0 データポート (`IO_TIMER_0` = 0x40) にカウント値の下位 8 ビットを出力
4. カウンタ 0 データポートにカウント値の上位 8 ビットを出力
5. 初期化完了メッセージを printk で表示

**呼び出し元:** `main()` (`i386/main.c`)

**注意点:**
- この時点では IRQ0 はまだマスクされている。`timer_start()` を呼ぶまで割り込みは発生しない。
- PIT はモード 3 (方形波) で動作する。出力波形は周期の前半が HIGH、後半が LOW となる。

### timer_intr

```c
void timer_intr(unsigned char apic, unsigned long delta);
```

**概要:** タイマー割り込みハンドラから呼び出されるタイマー処理関数。ティックカウンタの更新と VGA 表示、スケジューラのタイムアウト処理を行う。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `apic` | `unsigned char` | 割り込みを受けた CPU の識別子。0 = CPU 0 (BSP)、1 = CPU 1 (AP) |
| `delta` | `unsigned long` | タイマーティック増分値 (通常 1) |

**戻り値:** なし

**処理内容:**

1. `apic == 0` (CPU 0) の場合:
   - `timer_ticks` をインクリメント
   - `timer_ticks >= 1000000000UL` の場合、0 にリセット (オーバーフロー防止のラップアラウンド)
   - `vga_write_dec_at(3, 21, timer_ticks, 10, 0x0B)` でVGA 画面の行 3、列 21 にティック値を緑色 (属性 0x0B) で 10 桁幅表示
   - `sys_isig_tim(0)` を呼び出してシステム時刻を TIC_NUME (~17ms) 分進める
2. 全 CPU 共通: `sched_timeout(apic, delta)` を呼び出してスケジューラのタイムアウト処理を実行

**呼び出し元:**
- `c_intr_irq0()` (`i386/interrupt.c`) -- CPU 0 の PIT 割り込み (apic=0)

**注意点:**
- `timer_intr` を呼ぶのは CPU 0 の PIT ハンドラ (`c_intr_irq0`) のみ。CPU 1 の APIC タイマー (`c_intr_smp_timer1`) は EOI のみ送信し、`timer_intr` を呼ばない。これにより、デルタ符号化タイムアウトキューが PIT の ~17ms 周期 (HZ=60) だけで正確に減算される。
- `sched_timeout()` はタスクのスリープタイムアウト (`tslp_tsk`、`dly_tsk`) を処理するスケジューラ関数である。

### timer_stop

```c
void timer_stop(void);
```

**概要:** IRQ0 (PIT タイマー割り込み) をマスクして PIT 割り込みを停止する。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. `irq_mask_on(1)` を呼び出して IRQ0 をマスクする

**呼び出し元:** 現在未使用

**注意点:**
- `irq_mask_on()` の引数はビットマスクであり、ビット 0 が IRQ0 に対応する。引数 `1` は IRQ0 のマスクを意味する。

### timer_start

```c
void timer_start(void);
```

**概要:** IRQ0 (PIT タイマー割り込み) のマスクを解除して PIT 割り込みを有効にする。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. `irq_mask_off(1)` を呼び出して IRQ0 のマスクを解除する

**呼び出し元:** `smp_init()` (`i386/smp.c`) -- BSP が AP の起動完了を確認した後に呼び出される

**注意点:**
- タイマーの開始は `smp_init()` 内で `key_start()` と共に呼び出される。これは AP の初期化が完了するまで PIT 割り込みを遅延させるためである。
- `irq_mask_off()` の引数はビットマスクであり、ビット 0 が IRQ0 に対応する。引数 `1` は IRQ0 のアンマスクを意味する。

## 補足

- PIT は CPU 0 のみが使用する。CPU 1 のタイマーは Local APIC タイマーのみで動作する。
- 実効タイマー周期: 1193182 / 60 = 19886.37 カウント。整数切り捨てにより実際の周波数は約 60.0006 Hz となる。
- `timer_ticks` は `volatile` 宣言されており、割り込みハンドラとメインコードの間で安全に参照できる。
- VGA 表示位置 (3, 21) はカーネルステータス表示領域の一部として使用されている。
