# floppy.md

対象ファイル: `i386/floppy.c`, `i386/floppy.h`, `i386/floppyP.h`

## 概要

フロッピーディスクコントローラ (FDC) ドライバ。3.5 インチ 1.44MB フロッピーディスクの読み書きを DMA 転送で行う。FDC コマンドの発行、シーク、リキャリブレート、モーター制御、および ISA DMA (チャネル 2) のセットアップを含む。

現状ではデバッグ用の printk 出力が多数残されており、開発途上の状態である。

## 定数・マクロ

|------|------|------|
| `FDC_SEEK` | `0x0f` | シークコマンド |

| `FDC_WRITE` | `0xc5` | ライトデータコマンド (MT+MFM) |

### fdc_rw

```c
int fdc_rw(long addr, char* buf, int mode);
```

**概要:** 論理セクタアドレスを CHS に変換し、指定セクタの読み書きを DMA 転送で行う。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `addr` | `long` | 論理セクタアドレス (LBA) |
| `buf` | `char*` | 転送先/転送元バッファのアドレス |
| `mode` | `int` | 0 = リード、1 = ライト |

**戻り値:**

| 値 | 説明 |
|------|------|
| `0` | 成功 |
| `-1` | シークまたは結果エラー |

**処理内容:**

1. LBA→CHS 変換:
   - シリンダ `c = addr / (N_HEAD * N_SECTOR)`
   - ヘッド内オフセット `h2 = addr - c * (N_HEAD * N_SECTOR)`
   - ヘッド `h = h2 / N_SECTOR`
   - セクタ `s = h2 - h * N_SECTOR + 1` (セクタ番号は 1 始まり)
2. `fdc_start_motor()` でモーター起動
3. `fdc_seek(c, h, s)` で目標シリンダにシーク。失敗時はリキャリブレートして -1 を返す
4. DMA セットアップ:
   - リード時: `dma_setup(buf, DMA_READ)` + `fdc_out(FDC_READ)`
   - ライト時: `dma_setup(buf, DMA_WRITE)` + `fdc_out(FDC_WRITE)`
5. FDC コマンドパラメータ出力: ヘッド/ドライブ、シリンダ、ヘッド、セクタ、サイズ (N=2: 512 バイト)、セクタ数 (18)、GAP、DTL
6. `fdc_sleep()` で割り込み待ち
7. `fdc_result()` で結果フェーズを読み取り、エラーなら -1 を返す

**呼び出し元:** `fdc_read()` マクロ、`fdc_wriet()` マクロ

**注意点:**
- バッファアドレスは ISA DMA の制約により 16MB 以下でなければならない (addr.h の `FDC_BUFFER = 0x10000` を使用)。
- デバッグ printk が残されている。

### fdc_result

```c
int fdc_result(void);
```

**概要:** FDC の結果フェーズのバイトを読み取る。

**引数:** なし

**戻り値:**

| 値 | 説明 |
|------|------|
| `0` | 成功 (FDC がビジーでなくなった) |
| `ERR_STAT` (`-3`) | ステータスエラー |

**処理内容:**

1. 最大 `MAX_RESULTS` (15) 回ループ:
   - ステータスレジスタを読み取り、`MASTER` ビット確認
   - `DIRECTION` ビット確認 (FDC→CPU 方向)
   - データレジスタから結果バイトを読み取り (読み捨て)
   - 短いディレイ後にステータスレジスタを再確認
   - `CTL_BUSY` が 0 なら成功として 0 を返す
2. 最大回数を超えたら `ERR_STAT` を返す

**呼び出し元:** `fdc_rw()`

**注意点:**
- デバッグ printk が残されている。一部のエラー処理コードが `continue` により到達不能になっている。

### fdc_out

```c
int fdc_out(unsigned char cmd);
```

**概要:** FDC にコマンドまたはパラメータバイトを送信する。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `cmd` | `unsigned char` | 送信するコマンド/パラメータバイト |

**戻り値:**

| 値 | 説明 |
|------|------|
| `0` | 成功 |
| `-1` | エラー (リセット必要またはリトライ超過) |

**処理内容:**

1. `fdc_need_reset` がセットされていればエラーを返す
2. 最大 `MAX_RETRY` (5) 回ループ:
   - ステータスレジスタを読み取り、`MASTER | DIRECTION` を確認
   - FDC がコマンド受付可能 (`CTL_ACCEPTING`) ならデータレジスタにコマンドを出力して成功
   - 受付不可なら長いディレイ (100 万サイクルループ) 後にリトライ
3. リトライ超過時はエラーを返す

**呼び出し元:** `fdc_rw()`, `fdc_seek()`, `fdc_recalibrate()`, `fdc_reset()`

**注意点:**
- ビジーウェイトによるリトライを行うため、割り込みコンテキストでの使用は適さない。

**注意点:**

### fdc_sleep

```c
void fdc_sleep(int* val);
```

**概要:** FDC 割り込みをビジーウェイトで待機する。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `val` | `int*` | スリープフラグへのポインタ (`fdc_sleep_val`) |

**戻り値:** なし

**処理内容:**

1. `*val` を 1 にセット
2. `*val` が 0 になるまでビジーウェイトループ (割り込みで `fdc_intr` がクリアするまで)

**呼び出し元:** `fdc_rw()`, `fdc_seek()`, `fdc_recalibrate()`, `fdc_reset()`

**注意点:**
- ビジーウェイトによる待機であり、CPU を占有する。タスクのスリープ (`slp_tsk`) は使用していない。

### fdc_reset

```c
int fdc_reset(void);
```

**概要:** FDC をリセットし、スペシファイコマンドで動作パラメータを設定する。

**引数:** なし

**戻り値:** 常に `0`

**処理内容:**

1. 割り込み禁止 (`ccli()`)
2. `fdc_motor` と `fdc_goal` を 0 にクリア
3. DOR レジスタに 0 を出力 (FDC リセット)
4. DOR レジスタに `ENABLE_IF` (`0x0c`) を出力 (リセット解除、DMA/割り込み有効)
5. 割り込み許可 (`csti()`)
6. `fdc_sleep()` でリセット完了割り込みを待機
7. モーター起動
8. スペシファイコマンド出力: `FDC_SPECIFY`, `SPEC_1` (`0xcf`), `SPEC_2` (`0x10`)
9. `fdc_need_reset` を 0 にクリア

**呼び出し元:** 手動呼び出し (初期化時)

**注意点:**
- `ccli()`/`csti()` を使用しているため Ring 0 でのみ実行可能。

### dma_setup

```c
void dma_setup(unsigned long addr, int mode);
```

**概要:** ISA DMA チャネル 2 を 512 バイト転送用にセットアップする。

**引数:**

| パラメータ | 型 | 説明 |
|------------|------|------|
| `addr` | `unsigned long` | 転送バッファの物理アドレス |
| `mode` | `int` | DMA モード (`DMA_READ` = 0x46 または `DMA_WRITE` = 0x4a) |

**戻り値:** なし

**処理内容:**

1. 割り込み禁止 (`ccli()`)
2. フリップフロップクリア (`IO_DMA_STAT2`) にモードを出力
3. モードレジスタ (`IO_DMA_STAT1`) にモードを出力
4. アドレスレジスタ (`IO_DMA_ADDR`) に下位 8 ビットを出力
5. アドレスレジスタに上位 8 ビット (ビット 8-15) を出力
6. ページレジスタ (`IO_DMA_TOP`) にビット 16-19 を出力
7. カウントレジスタ (`IO_DMA_COUNT`) に (512 - 1) の下位 8 ビットを出力
8. カウントレジスタに (512 - 1) の上位 8 ビットを出力
9. 割り込み許可 (`csti()`)
10. マスクレジスタ (`IO_DMA_INIT`) に 2 を出力 (チャネル 2 アンマスク)

**呼び出し元:** `fdc_rw()`

**注意点:**
- ISA DMA は 16MB 境界を超えられない。バッファアドレスは 0x10000 (`FDC_BUFFER`) が想定されている。
- `ccli()`/`csti()` を使用しているため Ring 0 でのみ実行可能。

## 補足

- 3.5 インチ 1.44MB フロッピーの容量: 80 シリンダ x 2 ヘッド x 18 セクタ x 512 バイト = 1,474,560 バイト
- `fdc_wriet` マクロに typo がある (`fdc_write` が正しいスペル)。
- `fdc_sleep`/`fdc_intr` による待機方式はビジーウェイトであり、本来はセマフォやスリープベースの待機が望ましい。
- デバッグ用の printk が多数残っており、プロダクションコードとしては未完成の状態。
