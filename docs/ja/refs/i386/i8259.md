# i8259.md

対象ファイル: `i386/i8259.c`, `i386/i8259.h`, `i386/i8259P.h`

## 概要

Intel 8259A PIC (Programmable Interrupt Controller) のドライバ。マスター PIC とスレーブ PIC をカスケード構成で初期化し、割り込みベクタのリマッピングおよび EOI (End of Interrupt) 送信を行う。

本カーネルでは I/O APIC を使用せず、PIC 経由で外部 IRQ を CPU 0 のみに配送する構成を採用している。

## 定数・マクロ

### i8259P.h

| 定数 | 値 | 説明 |
|------|------|------|
| `EOI` | `0x20` | End of Interrupt コマンド。割り込み処理完了後にマスター・スレーブの両方に送信する |

### interrupt.h (関連定数)

| 定数 | 値 | 説明 |
|------|------|------|
| `VECT_IRQ0` | `0x80` | マスター PIC の IRQ0 に対応する割り込みベクタ番号 |
| `VECT_IRQ8` | `0x90` | スレーブ PIC の IRQ8 に対応する割り込みベクタ番号 |

### io.h (I/O ポートアドレス)

| 定数 | 値 | 説明 |
|------|------|------|
| `IO_I8259_M` | `0x020` | マスター PIC コマンドポート |
| `IO_I8259_MD` | `0x021` | マスター PIC データポート |
| `IO_I8259_S` | `0x0a0` | スレーブ PIC コマンドポート |
| `IO_I8259_SD` | `0x0a1` | スレーブ PIC データポート |

## 関数リファレンス

### i8259_init

```c
void i8259_init(void);
```

**概要:** マスター PIC とスレーブ PIC を初期化し、割り込みベクタをリマッピングする。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. **マスター PIC 初期化 (ポート 0x20/0x21)**
   - ICW1: `0x11` (エッジトリガ、カスケードモード、ICW4 使用)
   - ICW2: `VECT_IRQ0` (`0x80`) -- IRQ0-7 のベクタ開始番号
   - ICW3: `0x04` (IRQ2 にスレーブ接続)
   - ICW4: `0x0d` (バッファモード・マスター、8086 モード)

2. **スレーブ PIC 初期化 (ポート 0xa0/0xa1)**
   - ICW1: `0x11` (エッジトリガ、カスケードモード、ICW4 使用)
   - ICW2: `VECT_IRQ8` (`0x90`) -- IRQ8-15 のベクタ開始番号
   - ICW3: `0x02` (マスターの IRQ2 に接続)
   - ICW4: `0x09` (バッファモード・スレーブ、8086 モード)

3. **全 IRQ マスク (OCW1)**
   - マスターデータポートに `0xff` を出力 (全 IRQ マスク)
   - スレーブデータポートに `0xff` を出力 (全 IRQ マスク)

4. 初期化完了メッセージを printk で表示

**呼び出し元:** `main()` (`i386/main.c`)

**注意点:**
- QEMU では PIC 初期化後に全 IRQ がアンマスクされる場合があるため、ICW 設定後に必ず OCW1 で明示的にマスクする必要がある。
- 個別の IRQ のアンマスクは `irq_mask_off()` で後から行う。
- 割り込みベクタは x86 のデフォルト (IRQ0=0x08) からリマッピングされている。これは CPU 例外ベクタ (0x00-0x1F) との衝突を避けるためである。

### i8259_restore

```c
void i8259_restore(void);
```

**概要:** PIC を BIOS ブート時のデフォルト状態に復元する。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. **マスター PIC 復元**
   - ICW1: `0x11`
   - ICW2: `0x08` (BIOS デフォルト: IRQ0 = ベクタ 0x08)
   - ICW3: `0x04`
   - ICW4: `0x0d`

2. **スレーブ PIC 復元**
   - ICW1: `0x11`
   - ICW2: `0x70` (BIOS デフォルト: IRQ8 = ベクタ 0x70)
   - ICW3: `0x02`
   - ICW4: `0x09`

**呼び出し元:** 現在未使用 (カーネル終了時のリストア用途)

**注意点:**
- カーネルからリアルモードに戻る際など、BIOS 互換の割り込みベクタ設定が必要な場合に使用する。
- コメントに `i8250_restore` と typo がある (`i8259_restore` が正しい)。

### i8259_reenable

```c
void i8259_reenable(void);
```

**概要:** マスター・スレーブ両方の PIC に EOI コマンドを送信し、Local APIC の EOI も発行する。

**引数:** なし

**戻り値:** なし

**処理内容:**

1. マスター PIC コマンドポート (`0x20`) に EOI (`0x20`) を出力
2. スレーブ PIC コマンドポート (`0xa0`) に EOI (`0x20`) を出力
3. `smp_eoi()` を呼び出して Local APIC の EOI レジスタにも書き込む

**呼び出し元:** `c_intr_irq0` 〜 `c_intr_irq15` (全 IRQ ハンドラ)、`interrupt.c` 内の各割り込みハンドラ末尾

**注意点:**
- スレーブ PIC 経由の割り込み (IRQ8-15) ではスレーブとマスターの両方に EOI が必要だが、本実装では常に両方に送信している。
- `smp_eoi()` は Local APIC の EOI レジスタ (0xFEE000B0) に書き込みを行う。SMP 構成では APIC タイマー割り込みの完了にも必要である。

## 補足

- 本カーネルの割り込みベクタ割り当て:
  - IRQ0-7: ベクタ `0x80`-`0x87` (マスター PIC)
  - IRQ8-15: ベクタ `0x90`-`0x97` (スレーブ PIC)
  - syscall: ベクタ `0x99` (コールゲート)
- PIC はバッファモードで初期化されている (ICW4 のビット 3 が 1)。これはカスケード構成でマスター/スレーブを区別するための設定である。
