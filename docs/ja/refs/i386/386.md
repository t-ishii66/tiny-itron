# 386.c / 386.h / 386P.h

対象ファイル: `i386/386.c`, `i386/386.h`, `i386/386P.h`

## 概要

i386 のセグメントディスクリプタおよびゲートディスクリプタの設定を行うモジュール。
GDT (Global Descriptor Table) と IDT (Interrupt Descriptor Table) のエントリを
操作するための低レベル関数を提供する。

GDT はメモリアドレス `AL_GDT` (0x2000) に配置され、セグメントディスクリプタ (`seg_t`) の
配列としてアクセスされる。IDT のゲートディスクリプタも同じ構造体サイズ (8 バイト) のため、
`set_gate()` は GDT ポインタを `gate_t*` にキャストして利用する。

## 定数・マクロ

### セグメントタイプ (386.h)

| 定数 | 値 | 説明 |
|------|------|------|
| `ST_CODE` | `0x9a` | コードセグメント (実行可/読み取り可) |
| `ST_DATA` | `0x92` | データセグメント (読み取り/書き込み可) |
| `ST_STACK` | `0x96` | スタックセグメント (読み取り/書き込み可、expand-down) |
| `ST_TSS` | `0x89` | TSS (Task State Segment) ディスクリプタ |
| `ST_CALL` | `0x84` | コールゲート |

### ビット幅指定 (386.h)

| 定数 | 値 | 説明 |
|------|------|------|
| `_16BIT` | `0x00` | 16 ビットセグメント (D/B=0, G=0) |
| `_32BIT` | `0xc0` | 32 ビットセグメント (D/B=1, G=1) |

### ゲートタイプ (386.h)

| 定数 | 値 | 説明 |
|------|------|------|
| `GT_INTR` | `0x8e` | 32 ビット割り込みゲート (DPL=0, P=1) |
| `GT_TRAP` | `0x8f` | 32 ビットトラップゲート (DPL=0, P=1) |
| `GT_CALL` | `0x8c` | 32 ビットコールゲート |

### 386P.h の定義

`386P.h` には `CODE16`, `DATA16`, `STACK16`, `CODE32`, `DATA32`, `STACK32` が
マクロとして定義されているが、値は空 (未定義) である。将来の拡張用。

## 構造体・型

### seg_t (セグメントディスクリプタ)

```c
typedef struct seg {
    unsigned short  limit_l;    /* リミット下位 16 ビット */
    unsigned short  base_l;     /* ベースアドレス下位 16 ビット */
    unsigned char   base_m;     /* ベースアドレス中位 8 ビット (bit 16-23) */
    unsigned char   type;       /* タイプ + DPL + P ビット */
    unsigned char   limit_h;    /* リミット上位 4 ビット + フラグ (G, D/B, AVL) */
    unsigned char   base_h;     /* ベースアドレス上位 8 ビット (bit 24-31) */
} seg_t;
```

Intel i386 のセグメントディスクリプタ (8 バイト) に対応する構造体。
GDT 内の各エントリはこの構造体として表現される。

### gate_t (ゲートディスクリプタ)

```c
typedef struct gate {
    unsigned short  offset_l;   /* オフセット下位 16 ビット */
    unsigned short  sel;        /* セグメントセレクタ */
    unsigned char   count;      /* パラメータカウント (コールゲート用) */
    unsigned char   type;       /* タイプ + DPL + P ビット */
    unsigned short  offset_h;   /* オフセット上位 16 ビット */
} gate_t;
```

割り込みゲート、トラップゲート、コールゲートに使用される 8 バイト構造体。
IDT 内の各エントリはこの構造体として表現される。

## グローバル変数

### gdt

```c
seg_t* gdt = (seg_t*)AL_GDT;
```

GDT の先頭を指すポインタ。物理アドレス `0x2000` に配置されている。
`set_gdt()` と `set_gate()` の両方がこのポインタ経由で GDT/IDT にアクセスする。

## 関数リファレンス

### set_gdt

```c
void set_gdt(short n, unsigned long base, unsigned long limit,
             unsigned char type, unsigned char dpl, unsigned char gda)
```

**概要:** GDT の指定エントリにセグメントディスクリプタを設定する。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `n` | `short` | GDT エントリのバイトオフセット (例: `0x20`, `0x38`)。内部で 8 で割って配列インデックスに変換。 |
| `base` | `unsigned long` | セグメントのベースアドレス (32 ビット) |
| `limit` | `unsigned long` | セグメントのリミット (20 ビット) |
| `type` | `unsigned char` | セグメントタイプ (`ST_CODE`, `ST_DATA`, `ST_TSS` など) |
| `dpl` | `unsigned char` | 特権レベル (0-3)。type フィールドの bit 5-6 に設定される。 |
| `gda` | `unsigned char` | 粒度・ビット幅フラグ (`_16BIT` or `_32BIT`)。limit_h の上位ニブルに加算。 |

**戻り値:** なし (`void`)

**処理内容:**

1. `n` を 8 で割って配列インデックスに変換
2. `base` を 3 分割して `base_l` (bit 0-15), `base_m` (bit 16-23), `base_h` (bit 24-31) に設定
3. `limit` を `limit_l` (bit 0-15) と `limit_h` (bit 16-19) に分割して設定
4. `limit_h` に `gda` フラグを加算 (G ビットや D/B ビットを設定)
5. `type` に `dpl << 5` を OR して DPL を含む type フィールドを設定

**呼び出し元:** `tss_init()`, `start.s` (起動時の初期 GDT 設定)

**注意点:**
- `n` はセレクタ値 (バイトオフセット) を直接指定する。`SEL_K32_C` (0x20) なら GDT[4] に相当。
- `dpl` は `type` バイトの bit 5-6 に配置される。P (Present) ビットは `type` 定数に含まれる。

### set_gate

```c
void set_gate(short n, unsigned long offset, unsigned short sel,
              unsigned char type, unsigned char count)
```

**概要:** GDT 領域内にゲートディスクリプタを設定する。主に IDT エントリの設定に使用。

**引数:**

| 引数 | 型 | 説明 |
|------|------|------|
| `n` | `short` | エントリのバイトオフセット |
| `offset` | `unsigned long` | ハンドラ関数のオフセットアドレス (32 ビット) |
| `sel` | `unsigned short` | コードセグメントのセレクタ (通常 `SEL_K32_C`) |
| `type` | `unsigned char` | ゲートタイプ (`GT_INTR`, `GT_TRAP`, `GT_CALL`) |
| `count` | `unsigned char` | パラメータカウント (コールゲート用、割り込み/トラップゲートでは 0) |

**戻り値:** なし (`void`)

**処理内容:**

1. `n` を 8 で割って配列インデックスに変換
2. `gdt` ポインタを `gate_t*` にキャストしてゲート構造体としてアクセス
3. `offset` を `offset_l` (bit 0-15) と `offset_h` (bit 16-31) に分割して設定
4. `sel`, `count`, `type` を各フィールドに設定

**呼び出し元:** `set_idt()` (IDT エントリ設定の内部で間接的に使用される場合)

**注意点:**
- この関数は `gdt` ポインタを基準にオフセット計算するが、IDT は別のベースアドレス (`AL_IDT` = 0x2100) にあるため、`set_idt()` は `idt` ポインタを直接操作する方式を採用している。
- `set_gate()` はコールゲートの設定にも使用可能。

## 補足

### GDT レイアウト

GDT は `0x2000` から始まり、各エントリは 8 バイト。主要なセレクタ値:

| セレクタ | 用途 |
|----------|------|
| `0x08` (SEL_K16_C) | 16 ビットカーネルコード |
| `0x10` (SEL_K16_D) | 16 ビットカーネルデータ |
| `0x18` (SEL_K16_S) | 16 ビットカーネルスタック |
| `0x20` (SEL_K32_C) | 32 ビットカーネルコード |
| `0x28` (SEL_K32_D) | 32 ビットカーネルデータ |
| `0x30` (SEL_K32_S) | 32 ビットカーネルスタック |
| `0x38` (SEL_TSS0) | CPU 0 TSS |
| `0x40` (SEL_TSS1) | CPU 1 TSS |
| `0x48` | (未使用) |
| `0x50` | (未使用) |
| `0x58` (SEL_U32_C) | 32 ビットユーザーコード |
| `0x60` (SEL_U32_D) | 32 ビットユーザーデータ |
| `0x68` (SEL_U32_S) | 32 ビットユーザースタック |
| `0x70` (SEL_SYSCALL) | システムコール用コールゲート |

### セグメントディスクリプタの内部レイアウト

```
バイト 0-1: limit_l   (リミット bit 0-15)
バイト 2-3: base_l    (ベース bit 0-15)
バイト 4:   base_m    (ベース bit 16-23)
バイト 5:   type      (P + DPL + S + Type)
バイト 6:   limit_h   (G + D/B + 0 + AVL + リミット bit 16-19)
バイト 7:   base_h    (ベース bit 24-31)
```
